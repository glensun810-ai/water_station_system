# AI产业集群空间服务系统 - 快速启动指南

## 一、快速启动（推荐）

### Windows用户
双击运行 `start_services.bat`

### Mac/Linux用户
运行命令：
```bash
./start_services.sh
```

### 或直接使用Python
```bash
python start_services.py
```

## 二、自动完成的功能

✅ **自动检查端口占用** - 自动选择8000-8099范围内的可用端口
✅ **自动启动服务** - 依次启动水站、会议室、用餐三个服务
✅ **自动打开浏览器** - 自动打开系统首页
✅ **显示服务状态** - 实时显示服务运行状态
✅ **统一管理** - Ctrl+C 一键停止所有服务

## 三、访问地址

启动成功后，可通过以下地址访问：

### 首页和Portal
- 🏠 系统首页: http://localhost:8000/portal/index.html
- 📊 管理后台: http://localhost:8000/portal/admin/index.html

### 各服务
- 💧 水站管理: http://localhost:8000/water-admin/index.html
- 🏢 会议室: http://localhost:8000/meeting-frontend/index.html
- 🍽️ 用餐管理: http://localhost:8000/portal/admin/dining.html

### API文档
- 📚 Swagger UI: http://localhost:8000/docs
- 📚 ReDoc: http://localhost:8000/redoc

## 四、服务端口分配

启动器会自动选择可用端口，默认分配：
- 水站服务: 8000
- 会议室服务: 8001
- 用餐服务: 8002

如果端口被占用，会自动递增到下一个可用端口。

## 五、日志文件

服务日志保存在 `logs/` 目录：
- `logs/water_service.log` - 水站服务日志
- `logs/meeting_service.log` - 会议室服务日志
- `logs/dining_service.log` - 用餐服务日志

## 六、手动启动单个服务

如果只需要启动单个服务：

### 启动水站服务
```bash
cd Service_WaterManage/backend
python main.py
```

### 启动会议室服务
```bash
cd Service_MeetingRoom/backend
python main.py
```

### 启动用餐服务
```bash
cd Service_Dining/backend
python main.py
```

## 七、故障排查

### 端口被占用
如果提示端口被占用，启动器会自动选择其他端口，无需手动处理。

### Python依赖缺失
如果提示依赖缺失，运行：
```bash
pip install -r requirements.txt
```

### 浏览器未自动打开
手动在浏览器中访问显示的首页地址即可。

### 服务启动失败
1. 检查日志文件：`logs/*.log`
2. 确认Python版本：需要Python 3.8+
3. 确认依赖已安装：`pip list`

## 八、停止服务

在运行启动器的终端窗口中按 `Ctrl+C` 即可停止所有服务。

## 九、高级配置

### 修改默认端口范围
编辑 `start_services.py` 中的 `start_port` 参数。

### 修改自动打开浏览器
编辑 `start_services.py`，注释或删除 `webbrowser.open(primary_url)` 行。

### 修改服务配置
编辑各服务的 `config/settings.py` 文件。

## 十、生产环境部署

生产环境建议使用：
1. 使用Gunicorn或uWSGI
2. 配置Nginx反向代理
3. 使用systemd管理服务
4. 配置SSL证书
