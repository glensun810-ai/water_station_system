# 云服务器部署配置说明

## 问题诊断

### 当前状态
- ✅ 本地环境数据加载正常
- ❌ 云服务器API无响应（https://jhw-ai.com）

### 问题根源
数据库路径硬编码导致云服务器环境不匹配：
- `api_meeting.py` 和 `api_meeting_approval.py` 使用了本地相对路径
- 云服务器目录结构可能不同，导致找不到数据库文件

---

## 修复方案

### 1. 环境变量配置（推荐）

在云服务器上设置环境变量，指定数据库绝对路径：

```bash
# 在云服务器上执行（根据实际路径调整）
export MEETING_DB_PATH="/path/to/Service_MeetingRoom/backend/meeting.db"
export WATER_DB_PATH="/path/to/Service_WaterManage/backend/waterms.db"
```

或者在启动脚本中设置：

```bash
# 在启动服务的脚本中添加
MEETING_DB_PATH=/var/www/AIchanyejiqun/Service_MeetingRoom/backend/meeting.db
uvicorn Service_WaterManage.backend.main:app --host 0.0.0.0 --port 8000
```

### 2. 配置文件方式

创建 `.env` 文件：

```env
MEETING_DB_PATH=/path/to/Service_MeetingRoom/backend/meeting.db
WATER_DB_PATH=/path/to/Service_WaterManage/backend/waterms.db
```

## 云服务器检查步骤

### 1. 检查服务状态

```bash
# SSH登录到云服务器
ssh user@120.76.156.83

# 检查服务是否运行
ps aux | grep uvicorn
netstat -tlnp | grep 8000

# 检查日志
tail -f /var/log/service.log  # 或者项目的logs目录
```

### 2. 检查数据库文件

```bash
# 查找数据库文件位置
find /var/www -name "meeting.db" 2>/dev/null
find /var/www -name "waterms.db" 2>/dev/null

# 检查文件权限
ls -la /path/to/Service_MeetingRoom/backend/meeting.db
ls -la /path/to/Service_WaterManage/backend/waterms.db
```

### 3. 重启服务

```bash
# 停止旧服务
kill -9 <进程ID>

# 使用正确配置启动
cd /path/to/AIchanyejiqun
export MEETING_DB_PATH=/实际路径/Service_MeetingRoom/backend/meeting.db
uvicorn Service_WaterManage.backend.main:app --host 0.0.0.0 --port 8000 --reload
```
