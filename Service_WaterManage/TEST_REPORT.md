## 发现并修复的问题

### 问题 1: 缺少单个产品查询 API
**现象:** 前端无法获取单个产品详情
**修复:** 添加 `GET /api/products/{product_id}` 接口
**文件:** `backend/main.py`

### 问题 2: 用户切换后未重置产品选择
**现象:** 切换用户后，之前选择的产品可能不属于新用户
**修复:** 在 `selectUser()` 函数中重置 `selectedProduct` 为第一个产品
**文件:** `frontend/index.html`

### 问题 3: 数量验证缺失
**现象:** 允许提交数量为 0 或负数的领取请求
**修复:** 在 `record_pickup()` 函数中添加数量验证
**文件:** `backend/main.py`

### 问题 4: 路由顺序问题
**现象:** `/api/products/{product_id}` 路由未被正确注册
**修复:** 调整路由定义顺序，将 POST 路由放在 GET 单条路由之前
**文件:** `backend/main.py`
