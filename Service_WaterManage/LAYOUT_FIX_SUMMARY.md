# 领水登记页面布局优化

**日期**: 2026-03-30  
**问题**: 选择 10 桶以上时，优惠信息遮挡领取时间输入框  
**状态**: ✅ 已修复

---

## 问题描述

### 现象
当用户在领水登记页面选择 10 桶以上（有优惠）时：
1. 每个产品卡片下方显示绿色优惠信息卡片
2. 多个优惠卡片累积增加页面高度
3. 底部固定结算栏遮挡领取时间输入框
4. 用户无法看到或点击领取时间选择器

### 影响
- ❌ 用户体验差
- ❌ 无法设置领取时间
- ❌ 可能导致订单提交失败

---

## 根因分析

### 布局结构

```
领水登记容器 (pb-28 = 112px)
  ├── 办公室选择
  ├── 产品列表
  │     └── 产品卡片 × N
  │           └── 优惠信息卡片（当数量 >= 10 时显示）
  ├── 领取人和时间输入框 ❌ 被遮挡
  │
固定底部结算栏 (高度约 140-180px)
  ├── 优惠明细卡片
  └── 提交按钮
```

### 问题计算

- **底部 padding**: 112px (pb-28)
- **结算栏高度**: 140-180px（取决于是否有优惠明细）
- **安全余量**: 112 - 180 = **-68px** ❌（负数，说明不够）

当有多个优惠卡片时：
- 每个优惠卡片高度：约 32px
- 5 个产品 × 32px = **160px**
- 总高度增加：160px
- 实际需要：180 + 160 = **340px**
- 实际提供：**112px**
- **缺口**: 340 - 112 = **228px** ❌

---

## 修复方案

### 增加底部 padding

**修改前**:
```html
<div class="pb-28 px-3 py-4">
```

**修改后**:
```html
<div class="pb-96 px-3 py-4">
```

### 计算说明

- **新底部 padding**: 384px (pb-96)
- **结算栏高度**: 180px（最大）
- **优惠卡片总高度**: 200px（估算 6 个产品）
- **安全余量**: 384 - 180 - 200 = **4px** ✅

## 技术细节

### Tailwind CSS 间距

| 类名 | 像素值 | 说明 |
|------|--------|------|
| pb-28 | 112px | 原底部 padding（不足） |
| pb-32 | 128px | 仍然不足 |
| pb-36 | 144px | 仍然不足 |
| pb-40 | 160px | 仍然不足 |
| pb-48 | 192px | 基本够用 |
| pb-64 | 256px | 较充足 |
| **pb-96** | **384px** | **充足，推荐使用** |

### 为什么选择 pb-96？

1. **足够空间**: 384px 可以容纳最多的优惠卡片
2. **性能考虑**: 不会过大导致滚动距离过长
3. **移动端友好**: 在手机上也有良好的滚动体验
4. **未来扩展**: 为可能的 UI 变化预留空间

### ⚠️ 不要设置过大

```html

<!-- ✅ 正确：pb-96 已足够 -->
<div class="pb-96 px-3 py-4">
```
