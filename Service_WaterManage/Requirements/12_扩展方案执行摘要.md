## 🎯 一句话总结

将现有"水站管理系统"通过**最小改动**（文案配置化 + 数据模型扩展）升级为"企业综合服务管理平台"，支持会议室、VIP 餐厅、保洁等 10+ 种服务类型。

---

## 📊 核心价值

| 维度 | 现状 | 扩展后 |
|-----|------|--------|
| 服务类型 | 1 种（水） | 10+ 种 |
| 业务边界 | 单一领水 | 全企业服务 |
| 收入来源 | 水费 | 多元化服务收入 |
| 系统复用率 | - | 90%+ |
| 开发成本 | - | 低（主要是配置） |

---

## 🔧 核心改动（4 项）

### 1. 数据模型扩展
```sql
-- products 表增加 3 个字段
ALTER TABLE products ADD COLUMN service_type VARCHAR(50);
ALTER TABLE products ADD COLUMN resource_config TEXT;
ALTER TABLE products ADD COLUMN booking_required INT;

-- office_pickup 表增加 2 个字段
ALTER TABLE office_pickup ADD COLUMN service_type VARCHAR(50);
ALTER TABLE office_pickup ADD COLUMN time_slot VARCHAR(100);
```

### 2. 文案配置化
```javascript
// frontend/config.js
const APP_CONFIG = {
  concepts: {
    product: { singular: '服务', plural: '服务' },
    pickup: { singular: '服务记录', plural: '服务记录' }
  },
  serviceTypes: [
    { value: 'water', label: '饮用水', icon: '💧' },
    { value: 'meeting_room', label: '会议室', icon: '🏛️' },
    { value: 'vip_dining', label: 'VIP 餐厅', icon: '🍽️' },
    // ... 共 10 种
  ]
};
```

### 3. 单位体系扩展
```
原有单位：桶、瓶、件、提
新增单位：小时、半天、天、次、项、间、人次、平方米、套
```

### 4. 前端页面改造
- 领水记录 → 服务记录（文案替换）
- 商品管理 → 服务管理（文案 + 单位选项）
- 库存预警 → 资源预警（文案替换）
- 新增：服务预约日历（可选）

## 💡 关键设计决策

### ✅ 采用方案 A：扩展现有表
- 零数据库迁移成本
- 现有功能不受影响
- 开发工作量最小

### ✅ 文案配置化
- 统一管理所有文案
- 支持多语言扩展
- 业务人员可调整

### ✅ 渐进式扩展
- 先试点（水 + 会议室）
- 验证后推广
- 降低风险

### ✅ 保持兼容
- 水业务完全不受影响
- 新旧服务并存
- 平滑过渡
