### 2️⃣ 双模式业务系统

系统支持两种业务模式，满足不同用户需求：

| 特性 | 先用后付 (credit) | 先付后用 (prepaid) |
|------|------------------|-------------------|
| **支付时点** | 结算时支付 | 预定时支付 |
| **优惠策略** | 标准价格，无优惠 | 买 N 赠 M 优惠 |
| **适用场景** | 临时用水、短期需求 | 长期用水、稳定需求 |
| **领取方式** | 一次性领取 | 可分次领取 |
| **结算方式** | 批量结算 | 实时扣减 |
| **财务记录** | 待支付→已支付待确认→已结清 | 实时入账 |

**业务价值**:
- 先用后付：降低用户使用门槛，适合临时需求
- 先付后用：提前锁定收入，增加客户粘性

### 3️⃣ 优惠系统

#### 3.1 买赠优惠（买 N 赠 M）

| 产品 | 优惠配置 |
|------|----------|
| 桶装水 | 买 10 赠 1 |
| 瓶装水 | 买 20 赠 2 |
| 中桶水 | 买 15 赠 1 |

**优惠计算逻辑**:
- 按用户维度累计交易次数
- 每满足触发数量 N，赠送 M 个
- 赠送部分金额为 0

#### 3.2 优惠券系统

| 类型 | 说明 | 示例 |
|------|------|------|
| **折扣券** | 按比例打折 | 95 折券 |
| **满减券** | 满额减免 | 满 100 减 10 |

**优惠券特性**:
- 支持设置最低消费金额
- 支持设置最大优惠金额（防止过度优惠）
- 支持按产品范围限制
- 支持按业务模式限制（prepaid/credit）
- 支持有效期管理
- 支持发放数量限制
- 自动匹配最优优惠券

**核心 API**:
- `POST /api/coupons` - 创建优惠券
- `GET /api/coupons` - 获取优惠券列表
- `POST /api/coupons/issue` - 发放优惠券给用户
- `POST /api/coupons/use` - 使用优惠券
- `GET /api/coupons/calculate` - 计算最优优惠券

## 🎯 系统特色与亮点

### 1. 双模式业务架构
- 先用后付 + 先付后用两种模式并存
- 不同模式支持不同的优惠策略
- 用户可自主选择适合的 mode

### 2. 智能优惠系统
- 买赠优惠（买 N 赠 M）自动计算
- 优惠券自动匹配最优方案
- 支持多种优惠类型叠加

### 3. 办公室账户体系
- 支持按办公室管理预付账户
- 办公室充值、领水、结算全流程
- 多办公室管理支持

### 4. 完整的结算流程
```
用户领取 → 待支付 → 用户申请结算 → 已支付待确认 → 管理员确认收款 → 已结清
```

### 5. 细粒度权限控制
- 基于角色的访问控制（RBAC）
- 前后端双重权限验证
- 操作审计日志记录

### 6. 删除审计与恢复
- 软删除机制，数据不丢失
- 删除原因记录
- 回收站支持恢复

### 7. 响应式设计
- 移动端优先，适配手机扫码
- 桌面端侧边栏导航
- 移动端底部导航

### 8. 技术架构优势
- **前端**: Vue 3 Composition API + TailwindCSS
- **后端**: FastAPI + SQLAlchemy
- **数据库**: SQLite（可迁移至 PostgreSQL/MySQL）
- **认证**: JWT + bcrypt
