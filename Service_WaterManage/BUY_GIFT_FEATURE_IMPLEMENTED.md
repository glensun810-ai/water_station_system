## 🎯 业务需求

### 核心需求

在充值授信时，根据用户选择的产品和数量，**自动计算买赠优惠**,并在结算详情中清晰展示:

1. **买 10 赠 1 规则**:
   - 10 桶：送 1 桶
   - 11-19 桶：送 1 桶
   - 20-29 桶：送 2 桶
   - 30-39 桶：送 3 桶
   - 以此类推...

2. **计算公式**:
   ```
   赠送数量 = floor(购买数量 / trigger_qty) × gift_qty
   实付数量 = 购买数量 - 赠送数量
   实付金额 = 实付数量 × 单价
   ```

3. **展示要求**:
   - 优惠规则提示
   - 详细计算过程
   - 节省金额
   - 最终应付金额

### 1. 后端智能推荐算法增强

**文件**: `backend/api_unified_order.py`

#### 1.1 买赠计算函数

```python
def calculate_gift_quantity(quantity: int, promo_config: PromotionConfig) -> int:
    """
    计算赠送数量
    
    规则：每满足 trigger_qty 个，赠送 gift_qty 个
    """
    if not promo_config:
        return 0
    
    # 计算有多少个完整的周期
    trigger_cycles = quantity // promo_config.trigger_qty
    
    # 总赠送数量 = 周期数 × 每个周期赠送数量
    return trigger_cycles * promo_config.gift_qty
```

**示例**:
- 配置：买 10 赠 1
- 购买 25 桶：25 ÷ 10 = 2 个周期 → 赠送 2 × 1 = 2 桶
- 实付：25 - 2 = 23 桶

## ⚠️ 注意事项

### 1. 优惠配置要求

- `trigger_qty` 必须 > 0
- `gift_qty` 必须 >= 0
- `is_active` 必须为 1 (激活状态)

### 2. 边界情况处理

- **数量不足 trigger_qty**: 不赠送
- **刚好等于 trigger_qty**: 按配置赠送
- **超过但未达下一个周期**: 向下取整

### 3. 与其他优惠叠加

目前支持:
- ✅ 买赠优惠 + 优惠券 (双重优惠)
- ✅ 买赠优惠 + 智能推荐

## 📞 技术支持

### 常见问题

**Q1: 如何修改优惠配置？**  
A: 在数据库 `promotion_configs` 表中修改对应产品的配置。

**Q2: 为什么我的订单没有赠送？**  
A: 检查是否选择了"先付后用"模式，只有该模式享受买赠优惠。

**Q3: 优惠券和买赠能同时使用吗？**  
A: 可以！系统会先计算买赠，再应用优惠券折扣。
