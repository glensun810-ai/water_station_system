## 🎯 实施概述

本次实施完成了水站管理系统的**预付费功能**，支持管理员创建预付订单、用户查看预付余额并分次领取水品。

### 核心功能

1. **管理端**：创建预付订单、确认收款、查看订单列表
2. **用户端**：查看预付余额、领取水品、余量提醒

## 🔧 技术实现细节

### 数据库设计

#### prepaid_orders 表
```sql
CREATE TABLE prepaid_orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,              -- 购买用户
    product_id INTEGER NOT NULL,           -- 产品 ID
    total_qty INTEGER NOT NULL,            -- 预付总数量
    used_qty INTEGER DEFAULT 0,            -- 已使用数量
    unit_price DECIMAL(10,2) NOT NULL,     -- 单价
    total_amount DECIMAL(10,2) NOT NULL,   -- 总金额
    discount_amount DECIMAL(10,2) DEFAULT 0, -- 优惠金额
    payment_status VARCHAR(20) DEFAULT 'unpaid',
    payment_method VARCHAR(20) DEFAULT 'offline',
    payment_at DATETIME,
    created_by INTEGER NOT NULL,           -- 创建人（管理员）
    confirmed_by INTEGER,                  -- 确认人（管理员）
    note TEXT,
    is_active INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### prepaid_pickups 表
```sql
CREATE TABLE prepaid_pickups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL,             -- 关联订单
    pickup_qty INTEGER NOT NULL,           -- 领取数量
    picked_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    picked_by INTEGER NOT NULL,            -- 领取人
    note TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### API 列表

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| POST | /api/prepaid/orders | 创建预付订单 | 管理员 |
| GET | /api/admin/prepaid/orders | 获取订单列表（管理员） | 管理员 |
| GET | /api/prepaid/orders | 获取用户订单 | 用户 |

| POST | /api/prepaid/pickups | 创建领取记录 | 认证用户 |
| GET | /api/prepaid/pickups | 获取领取记录列表 | 认证用户 |

## 🚀 使用流程

### 管理员操作流程

1. 登录管理后台（admin.html）
2. 点击左侧"预付订单"菜单
3. 点击"新建预付订单"
4. 选择用户、产品、输入数量和单价
5. 设置优惠金额（如买 10 赠 1）
6. 选择支付方式（线下/在线/挂账）
7. 点击"创建订单"
8. 确认收到款项后，点击"确认"按钮

### 用户使用流程

1. 登录用户端（index.html）
2. 点击"我的预付"Tab
3. 查看预付余额和订单列表
4. 点击"领取"按钮
5. 输入领取数量
6. 点击"确认领取"
7. 系统自动扣减余额

## ⚠️ 注意事项

### 数据安全
- 预付款金额精确到分
- 所有操作需要登录认证
- 管理员操作需要权限验证

### 业务规则
- 领取数量不能超过剩余余额
- 订单确认后用户才能领取
- 余量不足 20% 时自动提醒
- 支持未使用订单退款

### 已知限制
- 管理端 UI 需要完整集成 JavaScript 逻辑
