**实施日期：** 2026-03-24  
**功能状态：** ✅ 已完成

## 一、功能概述

### 需求背景
后台管理中的交易记录管理需要实现删除功能，但必须限制权限：
- **仅超级管理员（super_admin）** 可以删除交易记录
- 普通管理员（admin）和员工（staff）无删除权限
- 删除操作需要记录审计日志
- 支持单条删除和批量删除
- 支持恢复误删记录

### 功能特性
- ✅ 权限控制：仅超级管理员可见删除按钮
- ✅ 单条删除：逐条删除交易记录
- ✅ 批量删除：一次性删除多条记录
- ✅ 软删除：数据不物理删除，可恢复
- ✅ 审计日志：记录所有删除操作
- ✅ 恢复功能：支持恢复误删记录

## 二、技术实现

### 后端 API（已存在）

#### 1. 删除交易记录
**接口：** `POST /api/admin/transactions/delete`

**请求参数：**
```json
{
  "transaction_ids": [1, 2, 3],
  "reason": "数据录入错误"
}
```

**权限验证：**
- 必须是 `admin` 或 `super_admin` 角色
- 需要登录验证（JWT Token）

**软删除机制：**
```python
transaction.is_deleted = 1
transaction.deleted_at = now
transaction.deleted_by = current_user.id
transaction.delete_reason = request.reason
```

**审计日志：**
```python
delete_log = DeleteLog(
    operator_id=current_user.id,
    operator_name=current_user.name,
    action="delete_transaction",
    target_type="transaction",
    target_ids=json.dumps(request.transaction_ids),
    reason=request.reason
)
```

#### 2. 恢复交易记录
**接口：** `POST /api/admin/transactions/restore`

**请求参数：**
```
transaction_ids=1&current_user_id=1
```

**权限要求：**
- 必须是 `admin` 角色

#### 3. 查询交易记录（支持软删除过滤）
**接口：** `GET /api/admin/transactions?include_deleted=false`

**参数说明：**
- `include_deleted=true` - 显示所有记录（包括已删除）
- `include_deleted=false` - 只显示未删除记录（默认）

#### 4. 查看删除日志
**接口：** `GET /api/admin/delete-logs?limit=50`

**返回数据：**
```json
[
  {
    "id": 1,
    "operator_name": "Admin",
    "action": "delete_transaction",
    "target_ids": "[1, 2, 3]",
    "reason": "数据录入错误",
    "created_at": "2026-03-24T10:30:00"
  }
]
```

**批量删除按钮：**
```html
<!-- 批量删除按钮（仅超级管理员） -->
<div v-if="selectedTransactions.length > 0 && user?.role === 'super_admin'">
    <button @click="showBatchDeleteConfirm">批量删除</button>
</div>
```

#### 2. 删除确认弹窗

```html
<div v-if="showDeleteModal" class="fixed inset-0 bg-black bg-opacity-50">
    <div class="bg-white rounded-2xl p-6">
        <!-- 标题 -->
        <h3>⚠️ 删除交易记录</h3>
        
        <!-- 密码验证 -->
        <input type="password" v-model="deleteForm.password" 
               placeholder="请输入管理员密码验证身份">
        
        <!-- 删除原因 -->
        <textarea v-model="deleteForm.reason" 
                  placeholder="请输入删除原因（选填）"></textarea>
        
        <!-- 提示信息 -->
        <ul>
            <li>• 删除操作为软删除，数据可在回收站恢复</li>
            <li>• 所有删除操作将被记录在审计日志中</li>
            <li>• 只有管理员才能执行删除操作</li>
        </ul>
        
        <!-- 操作按钮 -->
        <button @click="confirmDelete">确认删除</button>
    </div>
</div>
```

#### 3. 删除操作函数

```javascript
// 显示删除确认弹窗
const showDeleteConfirm = (transactionId) => {
    deleteForm.value = {
        transactionIds: [transactionId],
        password: '',
        reason: ''
    };
    showDeleteModal.value = true;
};

// 显示批量删除确认
const showBatchDeleteConfirm = () => {
    if (selectedTransactions.value.length === 0) {
        alert('请先选择要删除的交易记录');
        return;
    }
    deleteForm.value = {
        transactionIds: selectedTransactions.value,
        password: '',
        reason: ''
    };
    showDeleteModal.value = true;
};

// 确认删除
const confirmDelete = async () => {
    try {
        const storedToken = localStorage.getItem('token');
        const res = await fetch(`${API_BASE}/admin/transactions/delete`, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${storedToken}`
            },
            body: JSON.stringify({
                transaction_ids: deleteForm.value.transactionIds,
                reason: deleteForm.value.reason
            })
        });
        
        const data = await res.json();
        
        if (res.ok) {
            alert(data.message || '删除成功');
            closeDeleteModal();
            loadTransactions(); // 重新加载交易记录
        } else {
            alert(`删除失败：${data.detail || '未知错误'}`);
        }
    } catch (error) {
        alert('删除失败，请稍后重试');
    }
};
```

## 三、数据库设计

### Transaction 表软删除字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_deleted` | INTEGER | 删除标记：0-未删除，1-已删除 |
| `deleted_at` | DATETIME | 删除时间 |
| `deleted_by` | INTEGER | 删除人 ID（外键） |
| `delete_reason` | TEXT | 删除原因 |

### DeleteLog 表（审计日志）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键 |
| `operator_id` | INTEGER | 操作人 ID |
| `operator_name` | TEXT | 操作人姓名 |
| `action` | TEXT | 操作类型：delete_transaction |
| `target_type` | TEXT | 目标类型：transaction |
| `target_ids` | TEXT | 目标 ID 列表（JSON） |
| `reason` | TEXT | 删除原因 |
| `ip_address` | TEXT | 操作 IP |
| `created_at` | DATETIME | 操作时间 |

## 四、使用流程

### 场景 1：单条删除

1. 登录管理后台（使用超级管理员账号）
2. 进入"交易记录"标签

4. 点击"确认删除"

1. 勾选"显示已删除记录"（如有）
2. 找到已删除的记录（红色半透明）
3. 点击"恢复"按钮
4. 记录恢复正常状态

### 场景 4：查看删除日志

1. 访问删除日志 API

## 十、注意事项

- 删除权限仅限超级管理员
