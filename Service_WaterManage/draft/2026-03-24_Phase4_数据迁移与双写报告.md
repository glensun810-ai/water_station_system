### 1. 数据迁移脚本

#### 1.1 完整迁移工具

**文件路径**: `backend/migrate_full.py`

**核心类**: `DataMigration`

**迁移流程**:

```
1. 备份数据库 → 2. 创建表结构 → 3. 迁移数据 → 4. 验证结果
     ↓                ↓              ↓            ↓
  waterms_       5 个新表      4 类数据     一致性检查
  backup_*.db
```

**迁移数据类型**:

| 数据类型 | 源表 | 目标表 | 迁移策略 |
|---------|------|--------|---------|
| 用户账户 | `users.balance_credit` | `user_account` | 全量迁移 |
| 预付订单 | `prepaid_orders` | `account_wallet` | 聚合迁移 |
| 交易记录 | `transactions` | `transactions_v2` | 分批迁移 |
| 优惠配置 | `promotion_config` | `promotion_config_v2` | 双模式配置 |

**迁移代码示例**:

```python
class DataMigration:
    def run_full_migration(self, backup=True) -> Dict:
        # 1. 备份数据库
        if backup:
            self.backup_database()
        
        # 2. 创建表结构
        self.create_tables()
        
        # 3. 迁移数据
        self.migrate_users()           # 用户账户
        self.migrate_prepaid_orders()  # 预付订单
        self.migrate_transactions()    # 交易记录
        self.migrate_promotion_configs()  # 优惠配置
        
        # 4. 验证迁移
        verification = self.verify_migration()
        
        return {
            'success': verification.get('is_valid', False),
            'stats': self.stats,
            'verification': verification
        }
```

### 4. 回滚机制

#### 4.1 回滚策略

```
触发条件 → 停止迁移 → 恢复备份 → 验证恢复
   ↓          ↓          ↓          ↓
错误过多   回滚开关   从备份文件   检查数据
```

#### 4.2 回滚方法

```python
def rollback(self) -> bool:
    """从备份恢复数据库"""
    import shutil
    
    # 查找最近的备份
    backup_files = [f for f in os.listdir('.') 
                    if f.startswith('waterms_backup_')]
    
    if not backup_files:
        return False
    
    latest_backup = max(backup_files)
    shutil.copy(latest_backup, self.db_path)
    
    return True
```

#### 4.3 回滚 API

```python
@router.post("/rollback")
def rollback_migration():
    """回滚迁移（从备份恢复）"""
    migration = DataMigration()
    success = migration.rollback()
    
    return {
        "success": success,
        "message": "回滚成功" if success else "回滚失败"
    }
```
