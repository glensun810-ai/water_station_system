### P0-3: 消息通知机制

**文件：**
- `backend/main.py` (Notification 模型和 API)
- `frontend/notifications.html` (消息中心页面)
- `backend/migrate_notifications.py` (数据库迁移脚本)

#### 实现功能
1. **通知类型**
   - 💰 结算通知（用户申请、管理员确认）
   - ⚠️ 库存预警（库存低于阈值）
   - ℹ️ 系统通知
   - 🔔 提醒通知

2. **通知 API**
   - `POST /api/notifications` - 创建通知
   - `GET /api/notifications` - 获取通知列表
   - `POST /api/notifications/{id}/read` - 标记已读
   - `POST /api/notifications/mark-all-read` - 全部已读
   - `GET /api/notifications/unread-count` - 未读数量

3. **自动通知触发**
   - 用户提交结算申请 → 通知管理员
   - 管理员确认收款 → 通知用户
   - 库存低于阈值 → 通知管理员

4. **消息中心页面**
   - 消息列表展示
   - 按类型筛选（全部/未读/结算/预警/通知）
   - 未读消息高亮显示
   - 一键全部已读
   - 实时刷新（30 秒）

#### 数据库迁移
```bash
cd backend
python migrate_notifications.py
```

#### 使用方式
```
访问：http://localhost:8080/frontend/notifications.html
需要先登录
```

## 三、技术变更

### 数据库变更

**新增表：** `notifications`

```sql
CREATE TABLE notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    type TEXT DEFAULT 'info',
    is_read INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、注意事项

### 1. 数据库迁移
- 必须先运行 `migrate_notifications.py` 创建通知表
- 否则通知功能无法使用

### 2. Token 认证
- 管理端页面需要登录获取 Token
- Token 存储在 localStorage 中
- Token 有效期 24 小时

### 3. 跨域问题
- 前端需要访问后端 API（localhost:8000）
- 确保后端 CORS 配置正确
