## 🎨 设计理念

### 核心原则
1. **紧凑高效** - 在最小空间内展示最多信息
2. **视觉层级** - 通过色彩和尺寸突出关键数据
3. **直观操作** - 图标化按钮，减少文字依赖
4. **数据可视化** - 使用进度条展示用量趋势

## 🎯 关键优化点

### 1. 状态标识优化

**设计**:
```html
<span class="status-badge">
    <span class="status-indicator"></span>
    正常/不足/用完
</span>
```

**视觉效果**:
- ✅ **正常**: 绿色圆点 + 绿色背景
- ⚠️ **不足**: 琥珀色闪烁圆点 + 琥珀色背景
- ❌ **用完**: 红色圆点 + 红色背景

**改进**:
- 从纯文字 → 图标 + 文字
- 增加动态效果（不足时闪烁）
- 位置从独立行 → 与办公室并排

---

### 2. 数据展示优化

**三色卡片设计**:

| 数据 | 背景色 | 文字大小 | 文字颜色 |
|------|--------|---------|---------|
| 预留 | bg-slate-50 | text-base | text-slate-700 |
| 剩余 | 状态色 bg | **text-xl** | 状态色 text |
| 已用 | bg-slate-50 | text-base | text-slate-700 |

**设计巧思**:
- 剩余数量使用 **大号字体** (text-xl) 突出显示
- 根据状态使用不同背景色（绿/琥珀/红）
- 预留和已用使用中性色，形成对比

---

### 3. 用量进度条

**新增功能**:
```
使用率 50% ━━━━━━━━░░░░░░ 50%
```

**色彩编码**:
- 0-49%: 🟢 绿色 (健康)
- 50-79%: 🟡 琥珀色 (预警)
- 80-100%: 🔴 红色 (紧张)

**价值**:
- 一眼看出账户使用趋势
- 提前预警需要充值
- 视觉化比数字更直观

---

### 4. 操作按钮优化

**从文字到图标**:

| 操作 | 优化前 | 优化后 | 尺寸变化 |
|------|--------|--------|---------|
| 充值 | [充值] | [+] | 80px → 32px |
| 编辑 | [编辑] | [✏️] | 80px → 32px |
| 删除 | [删除] | [🗑️] | 80px → 32px |

**改进**:
- 总宽度：260px → 112px (↓57%)
- 使用通用图标，无需文字说明
- 增加 hover 和 active 效果
- 添加 title 提示

## 🔧 技术实现

### 辅助函数

```javascript
// 计算使用率
const getUsageRate = (acc) => {
    if (!acc || !acc.reserved_qty || acc.reserved_qty === 0) return 0;
    return Math.round((acc.used_qty / acc.reserved_qty) * 100);
};

// 使用率样式
const getUsageRateClass = (acc) => {
    const rate = getUsageRate(acc);
    if (rate >= 80) return 'text-red-600 font-bold';
    if (rate >= 50) return 'text-amber-600 font-medium';
    return 'text-green-600 font-medium';
};

// 进度条样式
const getUsageBarClass = (acc) => {
    const rate = getUsageRate(acc);
    if (rate >= 80) return 'bg-red-500';
    if (rate >= 50) return 'bg-amber-500';
    return 'bg-green-500';
};
```

### 动态样式绑定

```html
<!-- 边框颜色 -->
:class="acc.status === 'normal' ? 'border-green-200' : 
        (acc.status === 'low' ? 'border-amber-300' : 'border-red-300')"

<!-- 剩余数量背景 -->
:class="acc.status === 'normal' ? 'bg-green-50' : 
        (acc.status === 'low' ? 'bg-amber-50' : 'bg-red-50')"

<!-- 进度条宽度 -->
:style="{ width: getUsageRate(acc) + '%' }"
```

## 🎯 设计决策文档

### 决策 1: 为什么删除创建时间？

**原因**:
- 移动端场景下，用户更关注"还剩多少"
- 创建时间是历史信息，使用频率低
- 如需查看，可在编辑弹窗中查看

**权衡**:
- ✅ 节省空间：~20px 高度
- ❌ 损失信息：创建时间不可见

**结论**: 利大于弊，移动端优先展示核心信息。

---

### 决策 2: 为什么使用图标而非文字？

**原因**:
- 图标识别速度快于文字
- 国际通用，无需翻译
- 节省空间 (32px vs 80px)

**权衡**:
- ✅ 空间节省：57%
- ✅ 视觉简洁
- ❌ 学习成本：需记住图标含义

**结论**: 使用通用图标（+、✏️、🗑️），学习成本低。

---

### 决策 3: 为什么添加进度条？

**原因**:
- 数字不够直观（50 vs 80 哪个多？）
- 进度条一眼看出趋势
- 色彩编码增强预警

**权衡**:
- ✅ 视觉直观
- ✅ 提前预警
- ❌ 增加 ~30px 高度

**结论**: 进度条价值远超空间成本。
