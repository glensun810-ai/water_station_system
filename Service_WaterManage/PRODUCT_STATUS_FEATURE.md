# 产品状态管理功能文档

## 问题背景

在库存管理中，历史库存产品存在以下痛点：
1. **无法删除** - 因为有历史交易记录关联，删除会导致数据不一致
2. **持续告警** - 库存告急提醒持续显示在数据看板，干扰管理员注意力
3. **体验差** - 重要信息被淹没，影响日常管理效率

## 解决方案：产品停用/归档机制

通过引入 `is_active` 字段，实现产品状态管理：

### 产品状态

| 状态 | 说明 | 用户端 | 库存预警 | 管理端 |
|------|------|--------|----------|--------|
| **启用 (在售)** | 正常销售的产品 | ✅ 可见可领取 | ✅ 参与预警 | ✅ 默认显示 |
| **停用 (已归档)** | 历史/停售的产品 | ❌ 不可见 | ❌ 不参与预警 | ⚙️ 可选择显示 |

### 核心功能

#### 1. 停用产品
- 用户端自动隐藏，无法领取
- 数据看板不再显示库存预警
- 历史交易记录保留，可追溯
- 管理端可随时重新启用

#### 2. 启用产品
- 用户端恢复可见
- 重新参与库存预警
- 可正常领取

#### 3. 筛选查看
- 管理端默认只显示"在售"产品
- 勾选"显示已停用产品"可查看全部

## 技术实现

### 数据库变更

**表**: `products`  
**字段**: `is_active INTEGER DEFAULT 1`
- `1` = 启用/在售
- `0` = 停用/已归档

**迁移命令**:
```bash
cd backend
python migrate.py
```

### API 接口

#### 1. 获取产品列表（支持筛选）
```
GET /api/products?active_only=true
```
- `active_only=true`: 仅返回启用状态的产品（用户端使用）
- `active_only=false`: 返回所有产品（管理端使用）

#### 2. 切换产品状态
```
PUT /api/products/{product_id}/toggle
```

**响应示例**:
```json
{
  "product_id": 8,
  "is_active": 0,
  "message": "已停用"
}
```

#### 3. 库存预警（仅启用产品）
```
GET /api/admin/inventory-alert?threshold=10
```
- 自动过滤已停用产品

#### 4. 领取水（校验状态）
```
POST /api/record
```
- 停用的产品无法领取，返回错误：`该产品已停用，无法领取`

### 前端变更

#### 管理端 (`admin.html`)
- ✅ 新增"显示已停用产品"复选框
- ✅ 产品列表显示状态标签（在售/已停用）
- ✅ 新增"停用/启用"切换按钮
- ✅ 已停用产品半透明显示
- ✅ 操作前二次确认

#### 用户端 (`index.html`)
- ✅ 仅显示启用状态的产品
- ✅ 仅显示启用且有库存的产品

## 使用流程

### 场景 1: 停用历史产品

1. 进入"管理后台" → "库存管理"
2. 找到需要停用的产品
3. 点击"停用"按钮
4. 确认停用操作
5. ✅ 产品从用户端隐藏
6. ✅ 数据看板不再预警

### 场景 2: 重新启用产品

1. 进入"管理后台" → "库存管理"
2. 勾选"显示已停用产品"
3. 找到已停用的产品
4. 点击"启用"按钮
5. 确认启用操作
6. ✅ 产品恢复对用户可见

### 场景 3: 查看完整产品列表

1. 进入"管理后台" → "库存管理"
2. 勾选"显示已停用产品"
3. ✅ 查看所有产品（含已停用）
