# 📊 数据看板重新设计文档

## 设计理念

作为国际顶级产品经理，我基于**行动导向 (Action-Oriented)** 和 **洞察驱动 (Insight-Driven)** 的原则重新设计了数据看板。

### 核心设计原则

1. **有用 (Useful)** - 每个数据都应该指导行动
2. **易用 (Easy to Use)** - 一眼看到重点，一键跳转到操作
3. **好用 (Delightful)** - 视觉清晰，交互流畅，反馈及时

---

## 问题诊断（Before）

### 原有看板的痛点

| 问题 | 表现 | 影响 |
|------|------|------|
| **只有数据，没有洞察** | 显示"本月领取量 100"，但不知道趋势 | 管理员无法判断好坏 |
| **缺少行动指引** | 看到库存预警，但要跳转才能处理 | 增加操作成本 |
| **信息分散** | 待办事项分散在各个 tab | 无法一眼看到"今天要做什么" |
| **被动响应** | 等人来处理，没有主动提醒 | 容易遗漏重要事项 |
| **统计维度单一** | "本月领取量 100 桶/瓶"——不同规格混在一起，无实际指导意义 | 无法指导采购和库存管理 |

## 2025-03-24 核心指标模块优化

### 优化背景

**问题：** 原有"本月领取量"将不同规格的产品（18L 桶装水、500ml 瓶装水等）混在一起统计，导致：
- 管理员无法准确了解各类产品的实际用量
- 无法指导采购决策（该补桶装水还是瓶装水？）
- 数据看起来"不错"，但实际使用价值低

### 优化方案

**核心指标分为两大类：用量统计 + 金额统计**

#### ① 用量统计（按单位分类）

| 单位 | 本月用量 | 交易笔数 | 产品明细 |
|------|---------|---------|---------|
| 🪣 桶装水 | 91 桶 | 78 笔 | 18L: 73 桶，12L: 18 桶 |
| 🧴 瓶装水 | 12 瓶 | 12 笔 | 500ml: 12 瓶 |
| 📦 件装水 | 9 件 | 6 笔 | 12L 件装：1 件，390ML: 8 件 |

**价值：**
- ✅ 清晰展示不同规格产品的用量分布
- ✅ 指导采购：桶装水用量大，需要重点保障库存
- ✅ 支持多产品明细展示，信息完整

#### ② 金额统计（按状态分类）

| 状态 | 金额 | 笔数 | 说明 |
|------|------|------|------|
| 已结算 | ¥980 | - | 已收款，资金已到账 |
| 待结算 | ¥110 | 0 笔 | 待收款，需要跟进 |
| 已申请待确认 | ¥0 | 0 笔 | 用户已申请，等待管理员确认 |

**价值：**
- ✅ 清晰掌握资金回收情况
- ✅ 待结算金额醒目提示，督促跟进
- ✅ 支持环比增长趋势，洞察业务变化

### 技术实现

#### 后端 API 变更

**接口：** `GET /api/admin/dashboard/summary`

**新增字段：**
```json
{
  "usage_stats": {
    "by_unit": [
      {
        "unit": "桶",
        "quantity": 91,
        "transaction_count": 78,
        "products": {
          "桶装水 (18L)": 73,
          "中桶水 (12L)": 12,
          "12L 桶装水 (12L)": 6
        }
      },
      {
        "unit": "瓶",
        "quantity": 12,
        "transaction_count": 12,
        "products": {
          "瓶装水 (500ml)": 12
        }
      }
    ],
    "total_quantity": 112
  }
}
```

#### 前端展示变更

- 核心指标模块分为上下两层：
  - 上层：用量统计（按单位分卡片展示）
  - 下层：金额统计（3 个卡片：已结算、待结算、已申请待确认）
- 用量卡片支持显示产品明细（当同一单位有多个产品时）
- 单位显示优化：桶→🪣 桶装水，瓶→🧴 瓶装水，件→📦 件装水

## 功能模块详解

### ① 待办事项面板

**设计意图：** 一眼看到"今天要做什么"

| 卡片 | 数据来源 | 点击跳转 | 行动指引 |
|------|---------|---------|---------|
| 待确认结算 | `pending_tasks.applications_count` | 结算申请 tab | → 去处理 |
| 待补货产品 | `pending_tasks.low_stock_count` | 库存管理 tab | → 去补货 |
| 待提醒结算 | `pending_tasks.remind_count` | 提醒结算 tab | → 去提醒 |
| 异常监控 | `pending_tasks.abnormal_count` | 未来扩展 | → 查看 |

**视觉设计：**
- 渐变背景色（蓝/红/琥珀/紫）区分优先级
- 大数字突出显示
- Hover 阴影效果，暗示可点击
- "→ 去处理" 行动按钮

---

### ② 核心指标

**设计意图：** 不仅显示数据，还显示趋势

| 指标 | 环比对比 | 视觉反馈 |
|------|---------|---------|
| 本月领取量 | ↑12% (绿色) / ↓5% (红色) | 箭头 + 颜色 |
| 已结算金额 | ↑18% / ↓3% | 绿色表示健康 |
| 待结算金额 | 待结算笔数 | 琥珀色警示 |
| 已申请待确认 | 待确认笔数 | 蓝色提醒 |

**数据计算：**
```python
# 环比增长率
qty_growth_rate = (本月 - 上月) / 上月 * 100
```

---

### ③ 快捷操作

**设计意图：** 常用功能一键直达

| 按钮 | 功能 | 颜色 | 图标 |
|------|------|------|------|
| 补货登记 | 跳转库存管理 | 蓝色 | ➕ |
| 确认收款 | 跳转结算申请 | 绿色 | ✅ |
| 发送提醒 | 跳转提醒结算 | 琥珀色 | 🔔 |
| 查看报表 | 跳转交易记录 | 紫色 | 📊 |

---

### ④ 库存预警

**设计意图：** 突出问题，促进行动

- 红色渐变背景，视觉突出
- 显示产品名称、规格、当前库存、预警线
- "去补货" 快捷链接

---

### ⑤ 部门排行

**设计意图：** 激励 + 快速洞察

- TOP 5 部门展示
- 金银铜牌标识（🥇🥈🥉）
- 显示领取次数和总量
- 网格布局，移动端自适应

### 前端实现

#### 数据加载
```javascript
const loadDashboard = async () => {
    const [reportRes, alertsRes, summaryRes, quickStatsRes] = await Promise.all([
        fetch(`${API_BASE}/admin/report`),
        fetch(`${API_BASE}/admin/inventory-alert?threshold=10`),
        fetch(`${API_BASE}/admin/dashboard/summary`),
        fetch(`${API_BASE}/admin/dashboard/quick-stats`)
    ]);
    
    dashboardSummary.value = await summaryRes.json();
    quickStats.value = await quickStatsRes.json();
};
```

#### 点击跳转
```javascript
// 待办事项卡片点击跳转
@click="activeTab = 'applications'"  // 结算申请
@click="activeTab = 'inventory'"     // 库存管理
@click="activeTab = 'remind'"        // 提醒结算
```

## 设计总结

这次重新设计遵循了 **"数据 → 洞察 → 行动"** 的设计哲学：

1. **数据 (Data)** - 收集核心业务指标
2. **洞察 (Insight)** - 通过趋势对比提供洞察
3. **行动 (Action)** - 每个数据都指向可执行的操作

最终实现了一个**有用、易用、好用**的管理员工作台。
