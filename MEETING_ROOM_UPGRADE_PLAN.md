## 三、产品升级详细方案

### 3.1 P0级升级（核心商业闭环 - 2周）

#### 🎯 目标：建立完整商业闭环，实现从预约到支付到订单管理的完整流程

#### 📦 功能包清单

##### 1. **会议室详情页升级** ⭐⭐⭐⭐⭐
**当前问题**：用户无法充分了解会议室配置，影响决策质量  
**升级方案**：
```
页面结构：
- 【顶部】轮播图展示（3-5张实景图，支持自动播放）
- 【基础信息】名称、位置、容纳人数、面积、开放时段
- 【价格展示】会员价/非会员价，半小时/小时价格对比
- 【设备配置】图标化展示（投影仪✅ 视频会议✅ 白板✅ WiFi✅）
- 【使用规则】醒目展示（取消规则、迟到规则、卫生要求）
- 【时段选择器】优化版（绿色可预约、灰色已占、黄色我的预约）
- 【智能推荐】基于历史数据的时段推荐
- 【快捷操作】立即预约、收藏、联系管理员、查看导航
```

**实现要点**：
- 响应式图片加载（缩略图+原图懒加载）
- 设备图标动态渲染（根据facilities字段）
- 使用规则Modal弹窗（必读协议）
- 集成时间段选择器（保留现有灵活时段功能）

**API需求**：
```python
GET /api/meeting/rooms/{room_id}/detail
返回：{
  "room": {...},
  "images": ["url1", "url2", ...],
  "facilities_detail": [
    {"name": "投影仪", "icon": " projector", "available": true},
    ...
  ],
  "usage_rules": {...},
  "available_slots": [...]
}
```

##### 2. **支付系统集成** ⭐⭐⭐⭐⭐
**当前问题**：预约后无支付流程，无法形成商业闭环  
**升级方案**：
```
支付流程：
1. 提交预约 → 创建待支付订单（15分钟倒计时）
2. 跳转支付页面 → 选择支付方式
   - 微信支付（扫码/公众号支付）
   - 支付宝支付（扫码/网页支付）
   - 余额支付（企业账户）
   - 企业账户代扣（认证企业）
3. 支付成功 → 更新订单状态为"已支付/待使用"
4. 支付失败 → 订单保留5分钟可重新支付
5. 超时未支付 → 自动取消订单

支付页面设计：
- 订单信息卡片（会议室、时间、费用）
- 支付倒计时（15分钟，动态倒计时）
- 支付方式选择（图标化，hover高亮）
- 支付二维码展示（微信/支付宝）
- 支付状态轮询（前端每3秒查询支付状态）
- 支付成功跳转（订单详情页）
```

**技术方案**：
```python
后端API：
POST /api/meeting/payments/create
{
  "order_id": "xxx",
  "payment_method": "wechat|alipay|balance",
  "amount": 100.00
}
返回：{"payment_url": "...", "qr_code": "...", "order_id": "..."}

GET /api/meeting/payments/status/{payment_id}
返回：{"status": "pending|success|failed", "paid_at": "..."}

前端逻辑：
- 创建支付订单后跳转支付页面
- 支付页面展示二维码 + 倒计时
- 每3秒调用status接口查询支付状态
- 支付成功后跳转订单详情页
- 超时自动调用cancel接口取消订单
```

##### 3. **我的订单页面** ⭐⭐⭐⭐⭐
**当前问题**：用户无法查看历史订单、无法管理预约  
**升级方案**：
```
页面结构：
- 【顶部】订单状态筛选（全部、待支付、待使用、使用中、已完成、已取消）
- 【订单卡片】列表展示
  - 会议室名称 + 缩略图
  - 预约时间（日期 + 时段）
  - 订单状态（醒目徽章）
  - 费用金额
  - 快捷操作（支付/取消/续费/评价）
- 【分页】支持分页加载（每页10条）
- 【搜索】按会议室名称、预约编号搜索

订单状态定义：
- 待支付（pending_payment）：15分钟内需支付
- 待使用（confirmed）：已支付，等待使用
- 使用中（active）：当前正在使用
- 已完成（completed）：使用完毕
- 已取消（cancelled）：用户取消或超时取消
- 退款中（refunding）：申请退款中
```

**API需求**：
```python
GET /api/meeting/bookings/my-orders
参数：user_id, status, page, limit, search
返回：{
  "orders": [...],
  "total": 100,
  "page": 1,
  "limit": 10
}

POST /api/meeting/bookings/{id}/pay
POST /api/meeting/bookings/{id}/cancel
POST /api/meeting/bookings/{id}/extend
```

##### 4. **订单详情页** ⭐⭐⭐⭐
**当前问题**：无法查看订单完整信息  
**升级方案**：
```
页面内容：
- 订单编号（大号字体展示）
- 会议室信息（名称、位置、图片）
- 使用时间（日期、时段、时长）
- 费用明细（单价 × 时长 = 总费用）
- 支付信息（支付方式、支付时间、交易号）
- 取消/续费/改签记录（时间线展示）
- 使用规则入口（点击弹窗展示）
- 客服联系（一键拨打）
- 导航按钮（调用地图导航）

操作按钮：
- 待支付：立即支付、取消订单
- 待使用：取消订单、续费延长、查看导航
- 使用中：结束使用、续费
- 已完成：再次预约、评价会议、导出凭证
```

##### 5. **取消/退款流程** ⭐⭐⭐⭐
**当前问题**：无法取消预约、无法处理退款  
**升级方案**：
```
取消规则（系统可配置）：
- 提前2小时以上取消：全额退款
- 提前1-2小时取消：扣30%费用
- 提前1小时内取消：扣50%费用
- 已开始使用：不可取消，全额扣款

退款流程：
1. 用户申请取消 → 系统计算退款金额
2. 弹窗确认退款金额
3. 确认取消 → 自动退款到原支付方式
4. 退款处理时间：
   - 微信/支付宝：1-3天
   - 余额账户：即时到账
5. 发送退款通知（短信/邮件）

自动取消机制：
- 未支付订单：15分钟后自动取消
- 迟到15分钟：自动取消（释放时段）
```

**API需求**：
```python
POST /api/meeting/bookings/{id}/cancel
{
  "user_id": "xxx",
  "reason": "..."
}
返回：{
  "refund_amount": 70.00,
  "refund_rule": "提前1小时取消，扣30%",
  "cancel_time": "...",
  "refund_estimate": "1-3天"
}
```

2. 项目会议：批量预约多个会议室
