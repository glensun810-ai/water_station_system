# 水站管理系统 - 功能设计说明文档

## 1. 系统概述

### 1.1 系统简介

本系统是一套面向企业/机构的饮用水管理系统，主要功能包括：
- 员工自助领取桶装水
- 预付账户管理（充值、余额消费）
- 办公室级结算管理
- 管理后台（库存、产品、用户、结算等）
- 数据统计与报表

### 1.2 系统架构

```
┌─────────────────┐     ┌─────────────────┐
│  用户端 (index.html)  │     │  管理后台 (admin.html)  │
│  Vue 3 + Tailwind    │     │  Vue 3 + Tailwind       │
└────────┬────────┘     └────────┬────────┘
         │                      │
         └──────────┬───────────┘
                    │
         ┌──────────▼───────────┐
         │    后端 API 服务     │
         │   (FastAPI/uvicorn)  │
         └──────────┬───────────┘
                    │
         ┌──────────▼───────────┐
         │    SQLite 数据库     │
         └─────────────────────┘
```

## 2. 用户端 (index.html) 功能详解

### 2.1 核心功能模块

#### 2.1.1 领水登记功能

**功能入口**: 顶部Tab「领水登记」

**操作流程**:
1. **选择办公室** - 从常用/不常用两个分组中选择目标办公室
2. **选择产品及数量** - 展示所有在售产品，支持数量增减
3. **填写领取信息** - 选择或手动输入领取人、领取时间

**核心逻辑**:
- 预付余额消费：系统自动检测预付账户余额，领取时优先消耗预付余额
- 买赠优惠：满足促销阈值时自动计算赠送数量
- 提交API: `POST /api/user/office-pickup`

**关键字段**:
| 字段 | 说明 |
|------|------|
| office_id | 办公室ID |
| product_id | 产品ID |
| quantity | 领取数量 |
| pickup_person | 领取人姓名 |
| pickup_person_id | 领取人用户ID |
| pickup_time | 领取时间 |

#### 2.1.2 我的记录功能

**功能入口**: 顶部Tab「我的记录」

**功能特性**:
- **日期快捷筛选**: 今天/昨天/3天/1周/1月/2月/本月/上月
- **日期范围筛选**: 自定义开始日期和结束日期
- **状态筛选**: 待付款/待确认/已结清/全部
- **金额统计**: 各状态下的笔数和金额汇总

**操作功能**:
- **去付款**: 标记待付款记录为已付款状态
- **批量去付款**: 选中多条记录批量标记为已付款
- **导出功能**: 支持导出筛选结果或全部记录为CSV

**提交API**: 
- 单条付款: `POST /api/office-pickup/{id}/pay`
- 批量付款: `POST /api/office-pickups/batch-pay`

#### 2.1.3 结算管理功能

**功能入口**: 顶部Tab「结算」

**功能模块**:

1. **办公室选择** - 从下拉列表选择结算的办公室

2. **预付账户充值**
   - 点击预付余额卡片打开充值弹窗
   - 选择产品、填写充值桶数
   - 充值需要管理员审核

3. **结算申请**
   - 将待付款记录打包生成结算单
   - 结算单状态：pending(待付款) → applied(付款待确认) → confirmed(已结清)

4. **结算单列表**
   - 展示所有结算单
   - 支持状态筛选和时间筛选
   - 可展开查看明细

**关键API**:
- 充值申请: `POST /api/office-recharges`
- 结算申请: `POST /api/user/office-settlement`
- 结算单列表: `GET /api/user/office-settlements`

#### 2.1.4 预付账户功能

**显示位置**: 产品列表中展示各产品的预付余额

**余额来源**:
- 管理员手动充值
- 用户自助充值（需审核）

## 3. 管理后台 (admin.html) 功能详解

### 3.1 核心功能模块

#### 3.1.1 首页仪表板 (Dashboard)

**入口**: 侧边栏「首页」

**展示内容**:

1. **待办事项卡片**
   - 待确认结算申请数量
   - 待补货产品数量
   - 待提醒结算（长期未结算用户）
   - 异常监控数据

2. **核心指标**
   - 用量统计（按单位分类）
   - 金额统计（已结算/待付款/待确认）

3. **快捷操作**
   - 补货登记
   - 确认收款
   - 发送提醒
   - 查看报表

4. **库存预警** - 展示库存不足产品列表

5. **办公室排行** - 本月用水量TOP 5

#### 3.1.2 领水记录 (Office Pickups)

**入口**: 侧边栏「领水记录」

**功能特性**:

1. **多维度筛选**
   - 日期快捷筛选/自定义范围
   - 办公室筛选
   - 产品筛选
   - 状态筛选（待付款/待确认/已结清）
   - 领取人搜索

2. **批量操作**
   - 批量提醒付款
   - 批量确认收款
   - 批量删除（需权限）

3. **单条操作**
   - 提醒付款
   - 确认收款
   - 删除记录

4. **导出功能** - 支持导出筛选结果

**关键API**:
- 领取记录列表: `GET /api/office-pickups`
- 确认收款: `POST /api/office-pickup/{id}/confirm`
- 批量确认: `POST /api/office-pickups/batch-confirm`
- 提醒付款: `POST /api/office-pickup/{id}/remind`

#### 3.1.3 结算记录 (Settlement)

**入口**: 侧边栏「结算记录」

**功能模块**:

1. **结算申请管理**
   - 查看待确认的结算申请
   - 批量确认收款

2. **结算单管理**
   - 查看所有结算单
   - 按状态筛选
   - 确认结算

3. **自动结算**
   - 自动生成月度结算单

4. **催款提醒**
   - 向长期未结算用户发送提醒

**关键API**:
- 结算申请列表: `GET /api/admin/settlement-applications`
- 确认结算: `POST /api/admin/confirm-settlement`
- 结算单列表: `GET /api/admin/office-settlements`
- 发送提醒: `POST /api/admin/remind-unsettled`

#### 3.1.4 产品管理 (Products)

**入口**: 侧边栏「产品管理」

**功能模块**:

1. **产品列表**
   - 查看所有产品
   - 筛选在售/已停用

2. **产品操作**
   - 库存管理（直接输入库存数量）
   - 编辑产品信息
   - 启用/停用产品
   - 删除产品

3. **添加新产品**
   - 产品名称、规格、单价
   - 初始库存、单位
   - 买N赠M促销设置

**关键API**:
- 产品列表: `GET /api/products`
- 创建产品: `POST /api/products`
- 更新产品: `PUT /api/products/{id}`
- 更新库存: `PUT /api/products/{id}/stock`

#### 3.1.5 库存管理 (Inventory)

**入口**: 侧边栏「库存管理」

**功能模块**:

1. **库存预警**
   - 库存不足（库存=0）
   - 库存预警（库存≤预警阈值）
   - 库存正常

2. **快速补货**
   - 一键补货功能

3. **预警阈值设置**
   - 为每个产品设置预警阈值

**关键API**:
- 库存预警列表: `GET /api/admin/inventory-alert`
- 更新库存: `PUT /api/products/{id}/stock`

#### 3.1.6 预付账户 (Office Accounts)

**入口**: 侧边栏「预付账户」

**功能模块**:

1. **账户列表**
   - 按办公室/产品/类型/状态筛选

2. **账户统计**
   - 账户总数
   - 信用模式/预付模式数量
   - 可用桶数统计
   - 库存预警统计

3. **账户操作**
   - 新建账户
   - 编辑账户（预留桶数、剩余桶数）
   - 充值/扣减
   - 删除账户

**关键API**:
- 账户列表: `GET /api/admin/office-accounts`
- 创建账户: `POST /api/admin/office-accounts`
- 充值: `POST /api/admin/office-accounts/{id}/recharge`

#### 3.1.7 办公室管理 (Offices)

**入口**: 侧边栏「办公室管理」

**功能模块**:

1. **办公室列表**
   - 搜索、排序、筛选（常用/不常用）

2. **办公室操作**
   - 新建办公室
   - 编辑办公室
   - 设为常用/不常用
   - 启用/禁用
   - 删除

3. **批量操作**
   - 批量设置常用状态
   - 批量删除

4. **关联查看**
   - 查看办公室关联的用户

**关键API**:
- 办公室列表: `GET /api/offices`
- 创建办公室: `POST /api/offices`
- 更新办公室: `PUT /api/offices/{id}`
- 删除办公室: `DELETE /api/offices/{id}`
- 批量设置: `POST /api/offices/batch-common`

#### 3.1.8 用户管理 (Users)

**入口**: 侧边栏「用户管理」

**功能模块**:

1. **用户列表**
   - 搜索（用户名/办公室）
   - 角色筛选（超级管理员/管理员/员工）
   - 状态筛选（活跃/禁用）
   - 办公室筛选

2. **用户操作**
   - 新建用户
   - 编辑用户
   - 启用/禁用
   - 删除

3. **批量操作**
   - 批量启用
   - 批量禁用
   - 批量删除

4. **角色说明**
   - 超级管理员 (super_admin): 系统最高权限，不可删除
   - 管理员 (admin): 管理后台操作权限
   - 员工 (staff): 普通用户

**关键API**:
- 用户列表: `GET /api/users`
- 创建用户: `POST /api/users`
- 更新用户: `PUT /api/users/{id}`
- 删除用户: `DELETE /api/users/{id}`

#### 3.1.9 回收站 (Trash)

**入口**: 侧边栏「回收站」

**功能模块**:

1. **回收站列表**
   - 展示所有已删除的交易记录
   - 显示删除时间、删除人

2. **数据保留规则**
   - 删除记录保留30天，超期自动永久删除

3. **操作**
   - 恢复记录
   - 永久删除

**关键API**:
- 回收站列表: `GET /api/admin/transactions?is_deleted=1`
- 恢复记录: `POST /api/admin/transactions/restore`
- 永久删除: `POST /api/admin/transactions/permanent-delete`

## 5. 结算状态流转

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Pending    │────▶│   Applied    │────▶│  Confirmed   │
│   (待付款)    │     │ (付款待确认)  │     │  (已结清)     │
└──────────────┘     └──────────────┘     └──────────────┘
       ▲                    │
       │                    │
       └────────────────────┘
              用户付款
```

## 6. 预付/信用双模式

系统支持两种账户模式：

### 6.1 预付模式 (Prepaid)
- 用户先充值后消费
- 支持买N赠M促销活动
- 余额实时扣减

### 6.2 信用模式 (Credit)
- 先消费后结算
- 定期生成账单
- 管理员确认收款后结清

## 8. 移动端适配

系统全面支持移动端访问，采用以下优化策略：

- 响应式布局（桌面端/移动端自适应）
- 移动端底部导航栏
