# MVP方案 - 阶段一：核心体验优化（P0）

## 用户场景

### 场景1：用户灵活预约会议室
**痛点**：预约流程6-7步，时间段固定（仅3个选项），无法自定义时长，预约效率低（耗时3-5分钟）

**解决方案**：
- 简化流程至2-3步，提供时间选择器
- 支持自定义时间段（30分钟-8小时），可选时段07:00-22:00
- 实时冲突检测，避免时间段重叠
- 预约成功后生成详情页和二维码凭证

### 场景2：用户自主管理预约
**痛点**：无法查看预约历史，无法修改/取消预约，缺乏自主权，预约成功无凭证

**解决方案**：
- 新增"我的预约"页面，展示所有预约记录
- 支持按状态筛选（待确认/已确认/已完成/已取消）
- 允许修改时间/人数/主题，允许取消（会议前2小时）
- 生成独立详情页，包含二维码、分享链接

### 场景3：管理员安全高效管理
**痛点**：后台无登录验证，统计数据模拟（不准确），操作无日志

**解决方案**：
- JWT登录验证，角色权限体系（超管/会议室管理员/财务）
- 真实统计：使用率计算（实际使用/可用总时长）、预约数、收入
- 操作日志记录（操作人、时间、内容、结果）

## 功能清单（P0）

1. **灵活时间段选择**
   - 时间选择器（HTML5 input type="time"）
   - 时长验证：30分钟≤时长≤8小时
   - 实时冲突检测（查询已预约时段）
   - 快捷时段按钮（上午/下午/晚上）

2. **我的预约功能**
   - 预约列表页面（my_bookings.html）
   - 状态筛选器、日期范围筛选
   - 修改预约（check_modify权限）、取消预约（填写原因）

3. **预约详情页/凭证**
   - 独立详情页（booking_detail.html?id=）
   - 二维码生成（qrcode库）
   - 分享链接、打印功能

4. **管理员权限验证**
   - 登录页面（admin_login.html）
   - JWT Token生成/验证（2小时过期）
   - 角色权限表（admin_roles）
   - 操作日志表（admin_operation_logs）

5. **真实统计数据**
   - 使用率公式：（实际使用小时/可用总小时）×100
   - 预约统计：今日/本周/本月、自定义范围
   - 收入统计、趋势图表（ECharts）

## 验收标准

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 预约完成率 | 85%（↑25%） | 预约成功数/预约尝试数 |
| 预约耗时 | 1-2分钟（↓60%） | 用户操作时间统计 |
| 用户自主管理率 | 80%（↑80%） | 自主操作数/总操作数 |
| 测试覆盖率 | ≥80% | pytest覆盖率检测 |
| 统计准确性 | 100% | 对比实际数据验证 |
