# 会议室预约取消功能使用指南

**更新日期**: 2026-04-03  
**适用系统**: AI产业集群空间服务管理平台

---

## 一、功能概述

会议室预约取消功能允许用户在特定条件下取消已预约的会议室，旨在平衡用户灵活性与资源利用率。

---

## 二、取消规则详解

### 2.1 时间窗口说明

```
预约创建 ────────────── 前24小时 ─────── 前2小时 ────── 开始时间
            │              │              │             │
            │              │              │             │
         随时可取消      免费取消      需申请取消     不可取消
```

### 2.2 详细规则表

| 预约状态 | 距开始时间 | 用户权限 | 操作方式 | 说明 |
|---------|-----------|---------|---------|------|
| **待确认** | 任意时间 | ✅ 完全权限 | 直接取消 | 预约尚未确认，可随时取消 |
| **已确认** | ≥ 24小时 | ✅ 完全权限 | 直接取消 | 充足提前量，免费取消 |
| **已确认** | 2-24小时 | ⚠️ 有限权限 | 申请取消 | 需填写原因，系统审核 |
| **已确认** | < 2小时 | ❌ 无权限 | 不可取消 | 时间过近，无法取消 |
| **进行中** | - | ❌ 无权限 | 不可取消 | 会议进行中 |
| **已完成** | - | ❌ 无权限 | 历史记录 | 已结束的预约 |
| **已取消** | - | ❌ 无权限 | 已取消 | 已取消状态 |

## 三、操作步骤

### 3.1 查看我的预约

1. 访问 **我的预约** 页面：`http://localhost:8080/Service_MeetingRoom/frontend/my_bookings.html`
2. 页面顶部显示统计数据：
   - 总预约数
   - 待付款
   - 待确认收款
   - 已付款
3. 使用筛选器按状态查看：
   - 全部状态
   - 待确认
   - 已确认
   - 进行中
   - 已完成
   - 已取消

### 3.2 取消预约流程

#### 情况一：可直接取消（待确认 或 已确认+≥24小时）

1. 在预约列表中找到目标预约
2. 点击 **"取消预约"** 按钮（红色）
3. 在弹窗中：
   - 查看预约详情
   - 查看取消规则说明
   - **选择取消原因**（必填）
   - 填写详细说明（选填）
4. 点击 **"确认取消"**
5. 系统提示"预约已成功取消"

#### 情况二：需申请取消（已确认+2-24小时）

1. 点击 **"申请取消"** 按钮（橙色）
2. 弹窗显示：
   - "需申请取消，请填写取消原因"
3. 填写取消原因和详细说明
4. 提交申请
5. 等待管理员审核

#### 情况三：不可取消

- 按钮显示灰色文字 "不可取消"
- 鼠标悬停显示原因提示

## 五、常见问题 FAQ

### Q1: 为什么我不能取消预约？

**可能原因**：
1. 距开始时间不足2小时
2. 会议已经开始或已结束
3. 预约状态已完成或已取消

**解决方案**：
- 距开始<2小时：联系管理员协助处理
- 其他情况：查看预约状态，确认是否需要其他操作

### Q2: 取消预约后费用会退还吗？

**退还规则**：
- ✅ 提前≥24小时取消：全额退还
- ⚠️ 提前2-24小时：需管理员审核后决定
- ❌ 提前<2小时：不退还

### Q3: 取消后可以恢复预约吗？

**答案**: 不可以。取消操作不可逆，如需重新预约，请创建新的预约单。

### Q4: 如何查看历史取消记录？

**方法**:
1. 在"我的预约"页面
2. 筛选器选择"已取消"
3. 查看所有取消记录

### Q5: 取消申请提交后多久能审核？

**审核时间**:
- 工作日：2小时内
- 非工作日：24小时内
- 紧急情况：联系管理员

## 六、注意事项

### 6.1 用户须知

1. **提前规划**: 建议提前24小时以上取消，避免审核流程
2. **填写原因**: 取消原因有助于改善服务，请如实填写
3. **及时处理**: 临近会议时间，取消限制越多
4. **保留凭证**: 建议截图保存取消成功的提示

### 6.2 系统限制

- 每个预约只能取消一次
- 取消后时间段立即释放给其他用户
- 已付款预约取消后，退款需1-3个工作日处理
- 恶意频繁取消可能影响信用评级
