# 部署脚本使用说明

## 📁 脚本分类

### 🚀 部署脚本（生产环境）

#### 1. 首次完整部署
**文件**: `start_full_deploy.sh`  
**用途**: 首次部署到服务器，包含数据库  
**执行位置**: 本地电脑  
**命令**:
```bash
./start_full_deploy.sh
```

**适用场景**:
- 第一次部署到服务器
- 需要部署真实数据
- 需要完整初始化环境

---

#### 2. 增量更新部署
**文件**: `update.sh`  
**用途**: 更新代码，保护数据库  
**执行位置**: 本地电脑  
**命令**:
```bash
./update.sh
```

**适用场景**:
- 更新前端界面
- 更新后端功能
- 修复Bug
- **不会覆盖服务器数据库**

---

### 🖥️ 服务器端脚本（自动调用）

#### 3. 服务器完整部署
**文件**: `deploy_full.sh`  
**用途**: 在服务器上执行完整部署  
**执行位置**: 服务器  
**注意**: 由 `start_full_deploy.sh` 自动调用

---

#### 4. 服务器增量部署
**文件**: `deploy_incremental.sh`  
**用途**: 在服务器上执行增量更新  
**执行位置**: 服务器  
**注意**: 由 `update.sh` 自动调用

---

### 💻 开发脚本（本地开发）

#### 5. 启动所有服务
**文件**: `start_all.sh`  
**用途**: 本地开发启动所有后台服务  
**执行位置**: 本地电脑  
**命令**:
```bash
./start_all.sh
```

---

#### 6. 启动服务器
**文件**: `start_server.sh`  
**用途**: 本地开发启动API服务器  
**执行位置**: 本地电脑  
**命令**:
```bash
./start_server.sh
```

---

#### 7. 停止所有服务
**文件**: `stop_all.sh`  
**用途**: 本地开发停止所有后台服务  
**执行位置**: 本地电脑  
**命令**:
```bash
./stop_all.sh
```

---

## 🎯 快速选择指南

### 我应该用哪个脚本？

| 场景 | 使用脚本 |
|------|----------|
| 第一次部署到服务器 | `start_full_deploy.sh` |
| 更新代码（保护数据）| `update.sh` |
| 本地开发测试 | `start_all.sh` |
| 停止本地服务 | `stop_all.sh` |

## ⚠️ 重要提醒

### 首次部署 vs 增量更新

**首次部署 (`start_full_deploy.sh`)**
- ✅ 上传并部署数据库
- ✅ 安装所有依赖
- ✅ 配置Nginx和系统服务

**增量更新 (`update.sh`)**
- ❌ 不覆盖数据库（保护数据）
- ✅ 更新代码
- ✅ 更新依赖
- ✅ 重启服务

---

## 🔐 数据安全

### 增量更新的数据保护机制

1. **部署前自动备份**服务器数据库
2. **使用rsync排除**数据库文件
3. **临时保存**现有数据库
4. **代码更新后恢复**数据库
5. **双重保护**确保数据不丢失

## 📞 问题排查

### 部署失败怎么办？

1. 检查服务器连接：`ssh root@120.76.156.83`
2. 查看服务状态：`systemctl status waterms`
3. 查看错误日志：`tail -f /var/log/waterms/waterms-error.log`
4. 重启服务：`systemctl restart waterms`

### 数据库丢失了怎么办？

从备份恢复：
```bash
ssh root@120.76.156.83
ls /backup/waterms/database/
cp /backup/waterms/database/waterms_backup_最新时间戳.db /var/www/jhw-ai.com/backend/waterms.db
systemctl restart waterms
```
