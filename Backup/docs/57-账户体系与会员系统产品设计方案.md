## 一、设计目标与核心原则

### 1.1 产品愿景
**打造一个开放、包容、可持续增长的AI产业集群空间服务平台**，让内部员工和外部用户都能享受优质服务，通过会员体系和裂变机制实现用户价值最大化。

### 1.2 设计原则

#### 1. 极简主义原则
- **下拉菜单只做三件事**：身份识别 + 快速导航 + 账户管理
- **统计数据不在此展示**：所有业务数据统计集中在首页/工作台
- **功能入口清晰**：一级菜单不超过6项

#### 2. 渐进式信息披露
- **角色驱动**：不同角色看到不同功能
- **按需展开**：默认显示核心功能，高级功能折叠
- **上下文相关**：根据用户状态动态调整菜单项

#### 3. 用户价值导向
- **内部用户**：提升工作效率，获得分享权益
- **外部用户**：享受优质服务，会员专属权益
- **管理员**：高效管理工具，全局掌控能力

## 二、账户名称下拉菜单优化设计

### 2.1 设计理念调整

#### ❌ 之前的问题
```
下拉菜单承载过多信息：
- 用水统计、会议室使用统计等数据
- 办公室详情信息
- 业务操作入口

导致：
- 菜单臃肿，用户注意力分散
- 数据更新不及时，容易误导
- 核心功能被淹没
```

#### ✅ 优化后的设计
```
下拉菜单核心职责：
1. 身份识别（我是谁）
2. 快速导航（我要去哪）
3. 账户管理（我的账户）

统计数据移至：
→ 首页/工作台（集中展示，支持交互）
→ 专门的统计页面（深入分析）
```

```sql
-- 用户类型字段
ALTER TABLE users ADD COLUMN user_type VARCHAR(20) DEFAULT 'internal';

-- 会员信息表
CREATE TABLE memberships (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    membership_type VARCHAR(20) NOT NULL,  -- 'monthly', 'yearly'
    start_date TIMESTAMP NOT NULL,
    end_date TIMESTAMP NOT NULL,
    is_active INTEGER DEFAULT 1,
    auto_renew INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 邀请记录表
CREATE TABLE referrals (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    referrer_id INTEGER NOT NULL,  -- 邀请人ID
    referee_id INTEGER NOT NULL,   -- 被邀请人ID
    referral_code VARCHAR(20) UNIQUE NOT NULL,
    reward_status VARCHAR(20) DEFAULT 'pending',  -- pending, completed
    reward_details JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (referrer_id) REFERENCES users(id),
    FOREIGN KEY (referee_id) REFERENCES users(id)
);

-- 优惠券表
CREATE TABLE coupons (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    coupon_type VARCHAR(50) NOT NULL,  -- 'meeting_discount', 'water_coupon'
    discount_rate DECIMAL(3,2),  -- 0.80 表示8折
    discount_amount DECIMAL(10,2),  -- 减免金额
    is_used INTEGER DEFAULT 0,
    valid_from TIMESTAMP NOT NULL,
    valid_until TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 五、安全与风控

### 5.1 权限验证

```python
def check_user_permission(user_id: int, resource: str, action: str):
    """
    权限验证中间件
    
    参数：
    - user_id: 用户ID
    - resource: 资源类型（meeting_room, water, dining）
    - action: 操作类型（view, book, manage）
    """
    user = get_user(user_id)
    
    # 超级管理员：全部权限
    if user.role == 'super_admin':
        return True
    
    # 系统管理员：全部权限
    if user.role == 'admin':
        return True
    
    # 办公室管理员：管理权限
    if user.role == 'office_admin' and action in ['view', 'book', 'manage']:
        return check_office_admin_permission(user_id, resource)
    
    # 内部用户：使用权限
    if user.user_type == 'internal' and action in ['view', 'book']:
        return True
    
    # 外部用户：根据会员等级
    if user.user_type == 'external':
        membership = get_membership(user_id)
        return check_external_permission(membership, resource, action)
    
    return False
```
