## 结算状态流转

### 状态定义
- **pending**: 待付款 - 新创建的领水记录默认状态
- **applied**: 待确认 - 已提交结算申请，等待确认
- **settled**: 已结清 - 结算已完成

### 状态流转图
```
pending → applied → settled
   ↑         ↓
   └──── rejected
```

### 状态流转规则
1. **pending → applied**: 管理员提交结算申请
2. **applied → settled**: 管理员确认结算
3. **applied → pending**: 管理员拒绝结算

## 用户操作流程

### 结算申请流程
1. 在领水记录列表找到pending状态的记录
2. 点击"申请结算"按钮
3. 查看确认对话框中的详细信息
4. 点击"确定"提交申请
5. 系统更新状态为applied
6. 列表自动刷新

### 结算确认流程
1. 在领水记录列表找到applied状态的记录
2. 点击"确认结算"按钮
3. 查看确认对话框中的提示信息
4. 点击"确定"确认结算
5. 系统更新状态为settled
6. 列表自动刷新

### 结算拒绝流程
1. 在领水记录列表找到applied状态的记录
2. 点击"拒绝"按钮
3. 查看确认对话框中的提示信息
4. 点击"确定"拒绝申请
5. 系统更新状态回pending
6. 列表自动刷新
