# 统一登录系统设计文档

## 📋 设计理念

### 核心原则
1. **统一入口，智能分流** - 一个登录页面，根据用户角色自动跳转
2. **记忆功能** - 记住最近登录的账号，支持快速切换
3. **无缝体验** - 登录/退出流程流畅自然
4. **角色驱动** - 根据用户角色提供定制化体验

### 国际顶级产品对标
- **Google**: 统一登录页 + 账号切换
- **GitHub**: 记住最近账号 + 快速切换
- **Slack**: 工作空间切换 + 账号记忆
- **Notion**: 角色识别 + 智能路由

## 🏗️ 系统架构

### 1. 登录入口架构

```
/portal/login.html           # 统一用户登录页（推荐）
├── 快速登录（已保存账号）
├── 账号密码登录
└── 智能路由跳转

/portal/admin/login.html     # 管理端登录页（独立）
├── 管理员账号切换
├── 权限验证
└── 管理后台跳转
```

### 2. 用户角色体系

```javascript
角色层级：
- super_admin    // 超级管理员
- admin          // 系统管理员
- office_admin   // 办公室管理员
- user           // 普通用户
```

### 3. 智能路由规则

```javascript
登录后跳转逻辑：
├── super_admin / admin      → /portal/admin/index.html
├── office_admin             → /portal/admin/index.html
└── user                     → /portal/index.html 或之前的页面

退出登录跳转：
├── 管理员                   → /portal/admin/login.html
└── 普通用户                 → /portal/login.html
```

## 🔐 安全设计

### 1. 密码存储
- 本地存储使用简单加密（仅用于快速登录）
- 敏感操作需要重新输入密码
- 支持禁用密码记忆功能

### 2. Token管理
```javascript
localStorage存储：
- token           // 访问令牌
- user_id         // 用户ID
- userInfo        // 用户信息
- saved_accounts  // 已保存账号列表（加密）
```

### 3. 权限验证
- 管理端登录验证角色权限
- 非管理员无法登录管理后台
- API层二次验证权限

## 📊 数据流程

### 登录成功数据流
```
1. 用户提交登录表单
   ↓
2. 后端验证返回Token
   ↓
3. 获取用户详细信息
   ↓
4. 存储到LocalStorage
   - token
   - user_id
   - userInfo
   - saved_accounts
   ↓
5. 根据角色计算跳转URL
   ↓
6. 显示加载动画
   ↓
7. 跳转到目标页面
```

### 退出登录数据流
```
1. 用户点击退出
   ↓
2. 确认对话框
   ↓
3. 清除LocalStorage
   - token
   - userInfo
   - user_id
   ↓
4. 根据角色跳转到对应登录页
```

### 关键代码示例

#### 智能路由跳转
```javascript
const getRedirectUrl = (userInfo) => {
    const role = userInfo.role;
    
    if (['super_admin', 'admin', 'office_admin'].includes(role)) {
        return '/portal/admin/index.html';
    }
    
    const redirect = localStorage.getItem('login_redirect');
    if (redirect) {
        localStorage.removeItem('login_redirect');
        return redirect;
    }
    
    return '/portal/index.html';
};
```

#### 账号记忆
```javascript
const saveAccount = (userInfo) => {
    let accounts = JSON.parse(localStorage.getItem('saved_accounts') || '[]');
    
    // 移除已存在的账号
    accounts = accounts.filter(a => a.id !== userInfo.id);
    
    // 添加到开头
    accounts.unshift({
        id: userInfo.id,
        name: userInfo.name,
        role: userInfo.role,
        department: userInfo.department,
        timestamp: Date.now()
    });
    
    // 最多保存5个
    accounts = accounts.slice(0, 5);
    
    localStorage.setItem('saved_accounts', JSON.stringify(accounts));
};
```
