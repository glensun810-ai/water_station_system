# 生产环境部署政策

**版本**: 1.0  
**生效日期**: 2026-03-31  
**制定人**: 首席架构师

---

## 一、核心原则

1. **开发/生产环境严格隔离**
2. **生产环境变更必须经过审批**
3. **部署前必须备份**
4. **可回滚是部署的前提条件**

---

## 二、环境隔离规范

### 2.1 环境区分

| 环境 | 域名 | 端口 | 数据库 | 用途 |
|------|------|------|--------|------|
| 开发环境 | localhost | 8000-8004 | SQLite | 本地开发测试 |
| 生产环境 | jhw-ai.com | 80/443 | PostgreSQL | 线上服务 |

### 2.2 配置文件

```
# 开发环境
.env.development
- SECRET_KEY=dev-secret-key-change-in-production-2026
- DATABASE_URL=sqlite:///./waterms.db
- DEBUG=true

# 生产环境 (禁止提交到版本控制)
.env.production
- SECRET_KEY=<生产环境独立密钥>
- DATABASE_URL=postgresql://user:pass@host:5432/waterms
- DEBUG=false
```

### 2.3 前端配置

```javascript
// 禁止硬编码生产环境地址
const API_BASE = process.env.NODE_ENV === 'production' 
  ? 'https://jhw-ai.com/api' 
  : 'http://localhost:8000/api';
```

## 三、部署流程

### 3.1 部署前检查清单

- [ ] 代码已通过 Code Review
- [ ] 所有测试用例通过
- [ ] 已创建生产环境备份
- [ ] 已准备回滚方案
- [ ] 已获得负责人明确批准

### 3.2 部署命令

```bash
# ❌ 禁止直接部署
# python main.py --production

# ✅ 正确流程
# 1. 创建备份
./scripts/backup_production.sh

# 2. 验证备份
./scripts/verify_backup.sh

# 3. 等待批准 (必须由负责人明确指令)
# 4. 执行部署
./scripts/deploy_production.sh

# 5. 验证服务
./scripts/health_check.sh
```

### 3.3 备份策略

| 备份类型 | 频率 | 保留时间 | 存储位置 |
|----------|------|----------|----------|
| 数据库备份 | 每日 | 30 天 | 异地存储 |
| 代码备份 | 每次部署 | 永久 | Git 仓库 |
| 配置文件 | 每次变更 | 永久 | 加密存储 |

---

## 四、禁止事项

1. ❌ 禁止在开发环境使用生产数据库
2. ❌ 禁止将生产密钥提交到版本控制
3. ❌ 禁止未经批准直接修改生产环境
4. ❌ 禁止无备份直接部署
5. ❌ 禁止在生产环境进行调试
