## 一、设计目标

### 1.1 核心目标
- 统一的服务入口，用户只需记住一个URL
- 集中展示用户信息和各服务状态
- 提供快捷操作，提升用户体验
- 移动端优先，适配所有设备

### 1.2 用户场景
1. **首次访问**: 用户访问系统，看到统一的服务门户
2. **查看信息**: 用户查看自己的余额、待结算金额、免费时长等
3. **服务导航**: 用户点击服务卡片，跳转到对应服务页面
4. **快捷操作**: 用户快速完成领水、预约等常用操作

## 三、功能模块详细设计

### 3.1 顶部导航栏

**功能**:
- 显示系统Logo和名称
- 显示用户信息（头像、姓名、部门）
- 提供退出登录按钮

**交互**:
- 移动端：点击≡显示侧边菜单
- PC端：鼠标悬停显示用户下拉菜单

### 3.2 用户信息展示区域

**展示内容**:
- 水站余额（预付余额）
- 待结算金额（信用模式的未结算金额）
- 会议室免费时长（内部用户）
- 优惠券数量（可选）

**数据来源**:
- `GET /api/unified/user/{user_id}/balance` - 获取余额信息
- `GET /api/meeting/user/{user_id}/free-hours` - 获取免费时长

**更新频率**: 页面加载时获取，每隔5分钟自动刷新

### 3.3 服务导航卡片

#### 水站服务卡片
**功能入口**:
- 领水 → `/water/index.html`
- 我的余额 → `/water/user-balance.html`
- 套餐购买 → `/water/packages.html`
- 领水记录 → `/water/bookings.html`

**展示信息**:
- 服务图标：💧
- 服务名称：水站服务
- 快捷操作按钮
- 可选：显示余额快捷入口

#### 会议室服务卡片
**功能入口**:
- 预约会议室 → `/meeting/index.html`
- 我的预约 → `/meeting/my_bookings.html`
- 日历视图 → `/meeting/calendar.html`

**展示信息**:
- 服务图标：📅
- 服务名称：会议室服务
- 快捷操作按钮
- 可选：显示免费时长

#### 用餐服务卡片
**功能入口**:
- 预订餐食 → `/dining/index.html`（预留）
- 我的订单 → `/dining/orders.html`（预留）

**展示信息**:
- 服务图标：🍽️
- 服务名称：用餐服务
- 快捷操作按钮
- 标签：即将上线

### 3.4 快捷操作区域

#### 快速领水
**功能**: 在Portal首页直接完成领水操作，无需跳转

**流程**:
1. 选择办公室
2. 选择产品和数量
3. 提交订单
4. 显示成功提示

**使用组件**: OfficeSelector

#### 快速预约会议室
**功能**: 在Portal首页直接完成预约操作

**流程**:
1. 选择会议室
2. 选择日期和时间
3. 填写会议信息
4. 提交预约
5. 显示成功提示

**使用组件**: DatePicker

### 3.5 底部导航栏（移动端）

**导航项**:
- 首页：当前页面
- 服务：服务列表页
- 我的：个人信息页

**交互**: 固定在底部，支持安全区域适配

## 四、交互设计

### 4.1 页面加载
1. 显示加载动画
2. 并行获取用户信息、余额、免费时长
3. 渲染页面内容
4. 隐藏加载动画

### 4.2 服务卡片点击
1. 卡片点击反馈（缩放动画）
2. 跳转到对应服务页面
3. 传递用户信息（通过URL参数或localStorage）

### 4.3 快捷操作
1. 点击操作按钮
2. 显示操作面板（模态框或抽屉）
3. 完成操作后显示成功提示
4. 刷新用户信息

### 4.4 移动端侧边菜单
1. 点击≡按钮
2. 侧边菜单滑入
3. 显示用户信息和菜单项
4. 点击菜单项跳转
5. 点击遮罩层关闭菜单

## 五、响应式设计

### 5.1 断点定义
- 移动端：< 768px
- 平板：768px - 1024px
- PC端：> 1024px

### 5.2 布局适配
- 移动端：单列布局，底部固定导航
- 平板：双列布局，部分卡片并排
- PC端：三列布局，所有卡片并排

### 5.3 字体适配
- 移动端：标题20px，正文14px
- PC端：标题24px，正文16px

## 六、技术实现

### 6.1 技术栈
- Vue 3（Composition API）
- Tailwind CSS（已集成到设计系统）
- 公共组件库（阶段一已完成）

### 6.2 API接口
```javascript
// 获取用户信息
GET /api/user/info

// 获取水站余额
GET /api/unified/user/{user_id}/balance

// 获取待结算金额
GET /api/unified/settlement/summary?user_id={user_id}

// 获取免费时长
GET /api/meeting/user/{user_id}/free-hours

// 快速领水
POST /api/pickup

// 快速预约
POST /api/meeting/bookings
```

### 6.3 组件使用
```html
<!-- 用户信息卡片 -->
<StatsCard title="水站余额" :value="balance" icon="💧" color="primary" />

<!-- 办公室选择 -->
<OfficeSelector v-model="selectedOffice" @change="handleOfficeChange" />

<!-- 日期选择 -->
<DatePicker v-model="selectedDate" :show-quick-select="true" />
```
