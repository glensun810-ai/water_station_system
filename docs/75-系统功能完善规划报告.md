## 一、系统当前架构总览

### 1.1 部署架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        客户端浏览器                                      │
└───────────────────────────────┬─────────────────────────────────────────┘
                                │
                    ┌───────────▼───────────┐
                    │   Service_WaterManage │ (主后端 · 端口8000)
                    │   /backend/main.py    │
                    └───────────┬───────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
   ┌────▼────┐            ┌────▼────┐            ┌────▼────┐
   │ Portal  │            │  Water  │            │ Meeting │
   │ /portal │            │ /water  │            │/meeting │  ← 静态文件挂载
   └─────────┘            └─────────┘            └─────────┘
```

**关键发现**：
- 水站后端 (`Service_WaterManage/backend/main.py`) 是**唯一运行的主后端**
- 会议室后端 (`Service_MeetingRoom/backend/main.py`) 和用餐后端 (`Service_Dining/backend/main.py`) 是**独立服务，当前未集成**
- Portal、水站前端、会议室前端均通过主后端的静态文件挂载提供服务
