## 问题背景

用户反馈：超级管理员登录后，点击领水进入领水登记时，选择办公室没有展示出所有办公室，显示"您未设置所属办公室，请联系管理员设置"。

## 顺藤摸瓜诊断过程

### 第一步：找到错误源头

通过搜索错误信息"您未设置所属办公室"，找到了2个页面：
- `/portal/water/index.html` (Line 405)
- `/apps/water/frontend/index.html` (Line 418)

### 第二步：检查办公室加载逻辑

发现关键问题：
```javascript
// Line 1312-1315 (portal/water/index.html)
const [officesRes, productsRes] = await Promise.all([
    fetch(API_BASE + '/water/offices?is_active=true'),  // ❌ 没有认证头！
    fetch(API_BASE + '/water/products?active_only=true')
]);
```

**问题根源**：fetch调用没有带Authorization header，导致API返回401错误，offices数组为空。

### 第三步：检查权限判断逻辑

发现displayOffices逻辑问题：
```javascript
// Line 1036-1040 (原始代码)
displayOffices() { 
    if (this.userOffices.length > 0) {
        const userOfficeIds = this.userOffices.map(uo => uo.id);
        return this.offices.filter(o => userOfficeIds.includes(o.id) && ...);
    }
    return this.offices.filter(o => ...); 
}
```

**问题**：
- 超级管理员的`userOffices`是空的（localStorage中没有）
- 应该显示所有办公室，但`offices`数组为空（因为API调用失败）
- 导致显示"您未设置所属办公室"

### 第四步：检查API端点

测试API端点发现：
```
[无认证] curl /api/v1/water/offices → 401 Unauthorized
[有认证] curl /api/v1/water/offices -H "Authorization: Bearer $TOKEN" → 返回21个办公室
```

**结论**：API端需要认证，但前端没有传递token。

## 修复方案（上下游完整修复）

### 修复策略：四重保障

#### 1. 添加统一认证处理函数

```javascript
// Line 1289-1316 (portal/water/index.html)
checkAuth() {
    const token = localStorage.getItem('token');
    if (!token) {
        alert('请先登录');
        window.location.href = '/portal/admin/login.html?redirect=' + encodeURIComponent(window.location.pathname);
        return false;
    }
    return true;
}

handleApiError(response) {
    if (response.status === 401) {
        localStorage.removeItem('token');
        localStorage.removeItem('userInfo');
        alert('登录已过期，请重新登录');
        window.location.href = '/portal/admin/login.html?redirect=' + encodeURIComponent(window.location.pathname);
        return true;
    }
    return false;
}

getAuthHeaders() {
    const token = localStorage.getItem('token');
    const headers = { 'Content-Type': 'application/json' };
    if (token) headers['Authorization'] = `Bearer ${token}`;
    return headers;
}
```

#### 3. 修复displayOffices逻辑

```javascript
// Line 1035-1056 (修复后)
displayOffices() {
    const userRole = localStorage.getItem('user_role');
    
    // 超级管理员/管理员：显示所有办公室（按常用/不常用分类）
    if (userRole === 'admin' || userRole === 'super_admin') {
        return this.offices.filter(o => 
            this.officeTab === 'common' ? o.is_common === 1 : o.is_common !== 1
        );
    }
    
    // 普通用户/办公室管理员：显示用户所属办公室
    if (this.userOffices.length > 0) {
        const userOfficeIds = this.userOffices.map(uo => uo.id);
        return this.offices.filter(o => 
            userOfficeIds.includes(o.id) && 
            (this.officeTab === 'common' ? o.is_common === 1 : o.is_common !== 1)
        );
    }
    
    // 如果 userOffices 为空，显示后端返回的 offices（后端已过滤权限）
    return this.offices.filter(o => 
        this.officeTab === 'common' ? o.is_common === 1 : o.is_common !== 1
    );
}
```

**逻辑优化**：
- 根据用户角色判断显示逻辑
- 超级管理员/管理员：显示所有办公室
- 普通用户：显示所属办公室
- 权限过滤：前端+后端双重保障
