# AI产业集群空间服务系统 - 启动指南

## 🚀 快速启动

### 方式一：使用Python启动脚本（推荐）

```bash
python start_services.py
```

或直接执行：

```bash
./start_services.py
```

### 启动模式选择

启动脚本会显示以下选项：

```
1) 新架构模式 - 统一API网关（推荐）
   - 使用新架构 apps/main.py
   - 统一API路由 /api/v1/
   - 需要完成配置

2) 临时模式 - 从Backup恢复旧架构
   - 快速恢复服务
   - 功能完整
   - 稳定可用

3) 静态服务模式 - 仅前端页面
   - 无需后端配置
   - 快速预览
   - 功能受限

0) 退出
```

### 推荐启动流程

**场景1：首次使用或需要完整功能**
```
选择: 2) 临时模式
说明: 从Backup恢复旧架构，快速启动完整服务
```

**场景2：新架构已配置完成**
```
选择: 1) 新架构模式
说明: 使用统一API网关，享受新架构优势
```

**场景3：仅预览前端页面**
```
选择: 3) 静态服务模式
说明: 快速查看前端效果，无需后端API
```

## 📋 各模式详细说明

### 模式1：新架构模式

**适用场景**：
- 新架构已配置完成
- 需要使用统一API网关
- 准备生产部署

**前提条件**：
- ✅ `apps/main.py` 存在且配置正确
- ✅ `config/database.py` 已创建
- ✅ `config/settings.py` 已配置
- ✅ 数据库连接正常

**启动流程**：
1. 检查必要文件
2. 查找可用端口（默认8000）
3. 启动统一API网关
4. 健康检查
5. 打开浏览器

**访问地址**：
- Portal首页: `http://127.0.0.1:8000/portal/index.html`
- 水站服务: `http://127.0.0.1:8000/portal/water/index.html`
- 管理后台: `http://127.0.0.1:8000/portal/admin/login.html`
- API文档: `http://127.0.0.1:8000/docs`

**日志位置**：
```
logs/unified_api.log
```

---

### 模式2：临时模式（旧架构）

**适用场景**：
- 快速恢复服务
- 需要完整功能
- 新架构尚未配置完成

**工作原理**：
1. 检查旧架构是否存在
2. 如不存在，从Backup恢复
3. 启动旧架构服务

**前提条件**：
- ✅ `Backup/old_architecture_20260410_200812/` 存在

**启动流程**：
1. 检查旧架构文件
2. 如需要，从Backup恢复
3. 查找可用端口（默认8008）
4. 启动服务
5. 健康检查
6. 打开浏览器

**访问地址**：
- Portal首页: `http://127.0.0.1:8008/portal/index.html`
- 水站服务: `http://127.0.0.1:8008/water/index.html`
- 会议室服务: `http://127.0.0.1:8008/meeting-frontend/index.html`
- 管理后台: `http://127.0.0.1:8008/portal/admin/login.html`
- API文档: `http://127.0.0.1:8008/docs`

**日志位置**：
```
logs/water_service.log
```

---

### 模式3：静态服务模式

**适用场景**：
- 快速预览前端页面
- 无需后端功能
- 演示展示

**限制**：
- ❌ 无数据提交功能
- ❌ 无用户认证
- ❌ 无数据查询
- ✅ 可查看页面布局
- ✅ 可查看静态内容

**启动流程**：
1. 检查portal目录
2. 启动Python HTTP服务器
3. 打开浏览器

**访问地址**：
- Portal首页: `http://127.0.0.1:8000/portal/index.html`
- 水站服务: `http://127.0.0.1:8000/portal/water/index.html`

**日志位置**：
```
logs/static_server.log
```

## 🛠️ 故障排查

### 问题1：端口被占用

**错误信息**：
```
❌ 端口 8000 已被占用
```

**解决方案**：
```bash
# 查看占用端口的进程
lsof -i :8000

# 杀掉进程
kill -9 <PID>

# 或让脚本自动选择其他端口
# 脚本会自动查找可用端口
```

---

### 问题2：服务启动超时

**错误信息**：
```
❌ 服务启动超时（健康检查失败）
```

**解决方案**：
```bash
# 查看日志
cat logs/unified_api.log
cat logs/water_service.log

# 检查配置文件
ls -la config/
ls -la apps/

# 检查数据库连接
# 确保 PostgreSQL 或 SQLite 数据库可用
```

---

### 问题3：旧架构不存在

**错误信息**：
```
❌ Backup目录不存在
```

**解决方案**：
```bash
# 检查Backup目录
ls -la Backup/

# 如果没有old_architecture_*目录
# 说明旧架构未被迁移
# 请使用新架构模式
```

---

### 问题4：浏览器无法访问

**错误信息**：
```
This site can't be reached
```

**解决方案**：
```bash
# 1. 确认服务正在运行
ps aux | grep python

# 2. 确认端口监听
lsof -i :8000

# 3. 检查防火墙
# macOS: 系统偏好设置 > 安全性与隐私 > 防火墙

# 4. 尝试使用 127.0.0.1 而不是 localhost
# http://127.0.0.1:8000/portal/index.html
```

## 🔄 手动启动方式

### 启动新架构

```bash
# 1. 进入apps目录
cd apps

# 2. 设置端口
export PORT=8000

# 3. 启动服务
python main.py

# 4. 访问
# http://127.0.0.1:8000/portal/index.html
```

---

### 启动旧架构

```bash
# 1. 恢复旧架构（如需要）
cp -r Backup/old_architecture_20260410_200812/Service_WaterManage ./

# 2. 进入目录
cd Service_WaterManage/backend

# 3. 设置端口
export PORT=8008

# 4. 启动服务
python main.py

# 5. 访问
# http://127.0.0.1:8008/water/index.html
```

---

### 启动静态服务

```bash
# 1. 进入项目根目录
cd /path/to/AIchanyejiqun

# 2. 启动HTTP服务器
python3 -m http.server 8000

# 3. 访问
# http://127.0.0.1:8000/portal/index.html
```

## 📊 架构对比

### 新架构 vs 旧架构

| 特性 | 新架构 | 旧架构 |
|------|--------|--------|
| 服务入口 | 统一API网关 | 多个独立服务 |
| 端口使用 | 单端口 (8000) | 多端口 (8007, 8008...) |
| API路由 | `/api/v1/*` | `/api/*` |
| 前端入口 | 统一Portal | 分散的frontend |
| 配置管理 | 集中配置 | 分散配置 |
| 数据模型 | 统一模型 | 独立模型 |
| 扩展性 | 高 | 低 |
| 维护成本 | 低 | 高 |
| 稳定性 | ⚠️ 需完善 | ✅ 已验证 |

## 💡 最佳实践

### 开发环境

**推荐配置**：
- 使用新架构模式
- 开启调试模式
- 使用SQLite数据库
- 热重载配置

```bash
# 设置环境变量
export ENVIRONMENT=development
export DEBUG=true

# 启动服务
python start_services.py
# 选择 1) 新架构模式
```

---

### 生产环境

**推荐配置**：
- 使用新架构模式
- PostgreSQL数据库
- 关闭调试模式
- 配置日志轮转
- 设置进程守护

```bash
# 设置环境变量
export ENVIRONMENT=production
export DEBUG=false
export DATABASE_URL=postgresql://user:pass@host:5432/db

# 启动服务（建议使用 systemd 或 supervisor）
python start_services.py
```

---

### 临时演示

**推荐配置**：
- 使用临时模式（旧架构）
- 快速启动，功能完整

```bash
python start_services.py
# 选择 2) 临时模式
```
