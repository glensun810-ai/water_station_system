## 一、优化背景

### 1.1 当前问题

**问题1：多余的用户类型选择交互**
- 用户进入预约页面后，需要手动选择"内部员工"或"外部用户"
- 这增加了不必要的认知负担和操作步骤
- 用户体验不流畅，与国际标准不符

**问题2：统计维度混乱**
- 内部员工和外部用户都按"预约人"维度统计
- 缺少按"办公室"维度的统计（对企业内部管理不友好）
- 无法体现企业整体使用情况和成本归属

**问题3：身份识别滞后**
- 身份识别发生在预约流程中，而非系统登录时
- 导致后续流程依赖手动输入的信息
- 数据准确性依赖用户自觉性

### 1.2 国际标准参考

| 系统 | 用户识别方式 | 统计维度 | 优点 |
|-----|------------|---------|------|
| **Microsoft Outlook** | 登录账户自动识别 | 按部门统计 | 零额外操作，归属清晰 |
| **Google Calendar** | 登录账户自动识别 | 按团队统计 | 实时同步，数据准确 |
| **Airbnb** | 用户profile自动填充 | 按个人统计 | 一次填写，永久复用 |
| **WeWork** | 企业账户体系 | 按公司+团队统计 | 多维度统计，管理友好 |

## 二、优化方案

### 2.1 核心设计理念

**"登录即身份，身份即权限，权限即数据"**

- 登录账户承载所有身份信息
- 身份信息自动决定权限和统计维度
- 用户无需任何额外身份声明操作

### 2.2 用户身份模型重构

#### 原模型（有问题）：
```
用户登录 → 进入预约页面 → 手动选择用户类型 → 填写预约信息 → 提交
           (多余操作)      (手动输入)        (数据不准确)
```

#### 新模型（优化后）：
```
用户登录 → 身份自动识别 → 进入预约页面 → 自动填充信息 → 提交
           (系统自动)    (零操作)        (数据100%准确)
```

#### 身份识别逻辑：

| 用户角色 | 识别依据 | 统计维度 | 费用归属 | 权限 |
|---------|---------|---------|---------|------|
| **内部员工** | `role = 'user' or 'staff'` + `office_id != null` | 所属办公室 | 办公室账户 | 会员价 |
| **办公室管理员** | `role = 'office_admin'` + `office_id` | 所属办公室 | 办公室账户 | 审批权 |
| **系统管理员** | `role = 'admin' or 'super_admin'` | 全局统计 | 系统账户 | 全权限 |
| **外部访客** | `role = 'guest'` or `user_type = 'external'` | 个人 | 个人支付 | 非会员价 |

2. 选择用户类型（内部/外部）
3. 选择办公室（内部）或填写姓名电话（外部）

def recommend_spaces(user, date, time_slot):
    """
    智能推荐空间（参考Airbnb推荐系统）
    
    推荐权重：
    1. 常用空间权重：40%
    2. 可用性权重：30%
    3. 价格权重：20%
    4. 容量匹配权重：10%
    """
    
    # 1. 获取用户历史预约数据
    history = get_user_booking_history(user.id, limit=10)
    frequent_spaces = analyze_frequent_spaces(history)
    
    # 2. 查询可用空间
    available_spaces = query_available_spaces(date, time_slot)
    
    # 3. 计算推荐分数
    recommendations = []
    for space in available_spaces:
        score = 0
        
        # 常用空间加分
        if space.id in frequent_spaces:
            score += 40
        
        # 可用性加分
        if space.is_available:
            score += 30
        
        # 价格优惠（会员）
        if user.role == 'internal':
            effective_price = space.member_price
            score += 20
        else:
            effective_price = space.price
        
        # 容量匹配
        if space.capacity >= user.expected_attendees:
            score += 10
        
        recommendations.append({
            'space': space,
            'score': score,
            'price': effective_price,
            'is_favorite': space.id in frequent_spaces
        })
    
    # 4. 排序返回
    return sorted(recommendations, key=lambda x: x['score'], reverse=True)[:5]
```

@router.get("/stats/office/{office_id}")
def get_office_stats(
    office_id: int,
    start_date: date,
    end_date: date,
    current_user: User = Depends(get_current_user)
):
    """
    办公室维度统计（仅管理员和该办公室成员可查看）
    
    返回：
    - 办公室整体使用情况
    - 成员使用排行
    - 费用归属明细
    - 免费额度使用情况
    """
    
    # 权限检查
    if current_user.role not in ['admin', 'super_admin']:
        if current_user.office_id != office_id:
            raise HTTPException(403, "无权限查看该办公室统计")
    
    # 查询该办公室所有用户
    office_users = db.query(User).filter(User.office_id == office_id).all()
    
    # 查询这些用户的预约
    bookings = db.query(SpaceBooking).filter(
        SpaceBooking.user_id.in_([u.id for u in office_users]),
        SpaceBooking.booking_date >= start_date,
        SpaceBooking.booking_date <= end_date
    ).all()
    
    # 统计计算
    total_hours = sum([b.duration for b in bookings])
    total_cost = sum([b.actual_cost for b in bookings])
    
    # 用户排行
    user_stats = defaultdict(lambda: {'bookings': 0, 'hours': 0, 'cost': 0})
    for b in bookings:
        user_stats[b.user_id]['bookings'] += 1
        user_stats[b.user_id]['hours'] += b.duration
        user_stats[b.user_id]['cost'] += b.actual_cost
    
    top_users = sorted(
        user_stats.items(), 
        key=lambda x: x[1]['hours'], 
        reverse=True
    )[:10]
    
    return {
        "office_id": office_id,
        "office_name": get_office_name(office_id),
        "period": {
            "start": start_date,
            "end": end_date
        },
        "total_stats": {
            "bookings_count": len(bookings),
            "total_hours": total_hours,
            "total_cost": total_cost,
            "avg_cost_per_booking": total_cost / len(bookings) if bookings else 0,
            "users_count": len(office_users),
            "active_users_count": len(user_stats)
        },
        "top_users": [
            {
                "user_id": uid,
                "user_name": get_user_name(uid),
                "bookings": stats['bookings'],
                "hours": stats['hours'],
                "cost": stats['cost']
            }
            for uid, stats in top_users
        ],
        "free_hours_stats": {
            "allocated": sum([u.free_hours_monthly for u in office_users]),
            "used": calculate_free_hours_used(bookings),
            "remaining": calculate_free_hours_remaining(office_users)
        }
    }
```

## 五、数据模型调整

### 5.1 User模型扩展

```python
class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True)
    username = Column(String(50), unique=True, nullable=False)
    name = Column(String(100), nullable=False)
    phone = Column(String(20))
    email = Column(String(100))
    
    # 角色与身份
    role = Column(String(20), default='user')  # user/staff/office_admin/admin/super_admin/guest
    user_type = Column(String(20))  # internal/external（自动计算）
    
    # 所属办公室（关键！）
    office_id = Column(Integer, ForeignKey("offices.id"))
    office = relationship("Office", backref="users")
    
    # 会员信息
    member_level = Column(String(20), default='regular')  # regular/vip/super_vip
    member_since = Column(DateTime)
    member_expire = Column(DateTime)
    
    # 免费额度（每月）
    free_hours_monthly = Column(Integer, default=2)
    free_hours_used_current_month = Column(Integer, default=0)
    
    # 统计相关
    total_bookings = Column(Integer, default=0)
    total_hours = Column(Float, default=0.0)
    
    # 自动计算user_type
    @property
    def computed_user_type(self):
        """
        自动计算用户类型：
        - 有office_id → internal
        - role='guest' → external
        - 其他 → 根据role判断
        """
        if self.office_id:
            return 'internal'
        if self.role == 'guest':
            return 'external'
        if self.role in ['admin', 'super_admin', 'office_admin']:
            return 'internal'  # 管理员视为内部
        return 'external'
```

### 5.2 Booking模型扩展

```python
class SpaceBooking(Base):
    __tablename__ = "space_bookings"
    
    id = Column(Integer, primary_key=True)
    booking_no = Column(String(50), unique=True)
    
    # 用户信息（自动填充）
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    user = relationship("User", backref="bookings")
    
    # 用户类型（从user自动继承）
    user_type = Column(String(20))  # 自动填充：user.computed_user_type
    
    # 办公室归属（从user自动继承）
    office_id = Column(Integer, ForeignKey("offices.id"))
    office = relationship("Office", backref="bookings")
    
    # 预约人信息（兼容外部用户）
    user_name = Column(String(100))  # 自动填充：user.name
    user_phone = Column(String(20))  # 自动填充：user.phone
    
    # 空间信息
    resource_id = Column(Integer, ForeignKey("space_resources.id"))
    resource = relationship("SpaceResource", backref="bookings")
    resource_type = Column(String(20))  # meeting_room/auditorium/screen/booth/vip_dining
    
    # 时间信息
    booking_date = Column(Date, nullable=False)
    start_time = Column(String(10), nullable=False)
    end_time = Column(String(10), nullable=False)
    duration = Column(Float)  # 自动计算
    
    # 会议信息
    meeting_title = Column(String(200))
    attendees_count = Column(Integer, default=1)
    attendees_list = Column(Text)  # JSON格式的参会人列表
    
    # 状态
    status = Column(String(20), default='pending')
    
    # 费用
    original_cost = Column(Float, default=0.0)
    discount_rate = Column(Float, default=1.0)  # 自动计算：会员折扣
    final_cost = Column(Float, default=0.0)
    free_hours_applied = Column(Integer, default=0)
    
    # 支付
    payment_status = Column(String(20), default='pending')
    payment_method = Column(String(20))
    
    # 统计归属（关键！）
    cost_belong_to = Column(String(20))  # office/personal
    stats_dimension = Column(String(20))  # office_dimension/person_dimension
    
    @classmethod
    def create_from_user(cls, user, **kwargs):
        """
        从用户对象自动创建预约
        
        自动填充：
        - user_id
        - user_type
        - office_id
        - user_name
        - user_phone
        - discount_rate
        - stats_dimension
        """
        booking = cls(**kwargs)
        
        # 自动填充用户信息
        booking.user_id = user.id
        booking.user_type = user.computed_user_type
        booking.office_id = user.office_id
        booking.user_name = user.name
        booking.user_phone = user.phone
        
        # 自动计算折扣
        if user.computed_user_type == 'internal':
            booking.discount_rate = 0.8  # 会员价80%
            booking.cost_belong_to = 'office'
            booking.stats_dimension = 'office_dimension'
        else:
            booking.discount_rate = 1.0  # 非会员价100%
            booking.cost_belong_to = 'personal'
            booking.stats_dimension = 'person_dimension'
        
        return booking
```
