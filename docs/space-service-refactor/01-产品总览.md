**文档状态:** 正式规划

## 一、项目背景与目标

### 1.1 项目背景

当前AI产业集群空间服务系统的会议室管理模块仅支持会议室预约场景，功能单一且存在多处Bug和缺失。随着业务扩展，需要支持更多空间服务类型：

| 空间类型 | 当前状态 | 业务需求 |
|---------|---------|---------|
| 会议室 | 已实现 | 按小时预约，小型会议（10-20人） |
| 会场/多功能厅 | 未支持 | 按半天/全天预约，大型活动（50-500人） |
| 大堂大屏 | 未支持 | 按时段投放广告/展示内容 |
| 大堂展位 | 未支持 | 按天/周预约，产品展示/促销 |
| VIP餐厅 | 未支持 | 按餐段预约（午餐/晚餐时段），包厢服务 |

### 1.2 项目目标

**核心目标:** 构建一个通用、灵活、可扩展的空间服务预约管理平台

**具体目标:**

1. **统一空间服务模型** - 支持多种空间类型的统一管理
2. **灵活定价策略** - 支持按小时/半天/全天/餐段/周等多种计价单位
3. **标准化预约流程** - 建立符合国际标准的预约审批支付闭环
4. **修复现有Bug** - 解决当前会议室模块的所有问题
5. **补充缺失功能** - 实现消息通知、修改预约、日历同步等关键功能

## 二、产品定位

### 2.1 产品愿景

打造AI产业集群的"空间资源调度中枢"，成为企业空间服务管理的标杆平台，对标Microsoft Outlook会议室管理、Google Calendar资源预订、WeWork空间预约系统。

### 2.2 目标用户

| 用户类型 | 用户画像 | 核心需求 |
|---------|---------|---------|
| **普通员工** | 企业内部用户 | 快速预约会议室，查看预约状态，接收提醒 |
| **部门主管** | 有审批权限 | 审批下属预约，查看部门使用统计 |
| **行政管理员** | 空间管理员 | 管理空间资源，处理预约申请，配置定价策略 |
| **财务人员** | 结算管理员 | 处理费用结算，导出财务报表 |
| **访客/外部用户** | 外部租户 | 预约空间服务，在线支付，获取发票 |
| **企业管理层** | 决策者 | 查看整体使用率，优化资源配置 |

### 2.3 核心价值主张

1. **对于用户:** 一站式预约，实时状态，智能推荐
2. **对于管理者:** 全面管控，数据洞察，效率提升
3. **对于企业:** 资源优化，成本透明，服务升级

## 四、核心特性规划

### 4.1 空间服务类型管理

**设计原则:** 类型可扩展，配置灵活化

| 特性 | 说明 | 实现方式 |
|------|------|---------|
| **空间类型定义** | 支持会议室、会场、大屏、展位、VIP餐厅等 | 枚举+自定义扩展 |
| **时间单位配置** | 每种类型可配置计价单位（小时/半天/全天/餐段/周） | 系统配置表 |
| **容量范围配置** | 不同空间类型的人员容量范围 | 类型属性配置 |
| **设备设施管理** | 每种类型可配置标准设备清单 | 设备模板库 |
| **审批流程配置** | 不同类型可配置不同审批流程 | 流程引擎 |

### 4.2 预约流程标准化

**国际标准对比:** Microsoft Outlook + Google Calendar + WeWork

```
预约创建流程（国际标准版）:

用户登录/认证
    ↓
选择空间类型 → 查看可用空间列表
    ↓
选择具体空间 → 查看实时可用时段
    ↓
选择时段 → 智能推荐备选时段（冲突时）
    ↓
填写预约信息 → 会议主题、参会人数、参会人邀请
    ↓
费用计算 → 显示预估费用、优惠折扣
    ↓
提交预约 → pending状态
    ↓
审批流程（可选）→ 审批通过/拒绝
    ↓
支付确认 → 在线支付/线下支付确认
    ↓
预约确认 → 发送确认通知、日历邀请
    ↓
即将开始提醒 → 提前15分钟/1小时提醒
    ↓
签到/使用 → 可选签到功能
    ↓
完成/评价 → 服务评价反馈
```

### 4.3 定价策略引擎

**设计原则:** 灵活配置，规则可叠加

| 定价维度 | 配置项 | 示例 |
|---------|--------|------|
| **基础定价** | 按时间单位的基础价格 | 会议室: 50元/小时 |
| **时段定价** | 不同时段不同价格 | 工作时间/非工作时间 |
| **会员定价** | 会员等级折扣 | VIP会员: 80%价格 |
| **批量优惠** | 批量预约折扣 | 连续预约4小时以上: 90% |
| **节假日定价** | 特殊日期价格调整 | 节假日: 120% |
| **优惠活动** | 促销活动折扣 | 新用户首单: 50% |
| **免费额度** | 每月免费使用时长 | 内部员工: 2小时免费 |

### 4.4 通知服务体系

| 通知类型 | 触发时机 | 通知渠道 | 通知内容 |
|---------|---------|---------|---------|
| **预约创建通知** | 预约提交成功 | Email+短信+站内信 | 预约详情、待审批提示 |
| **审批通过通知** | 审批通过 | Email+短信+站内信 | 预约确认、支付提醒 |
| **审批拒绝通知** | 审批拒绝 | Email+站内信 | 拒绝原因、建议备选时段 |
| **支付成功通知** | 支付完成 | Email+短信 | 支付凭证、预约确认 |
| **即将开始提醒** | 提前15/30/60分钟 | 短信+站内信 | 空间位置、参会人提醒 |
| **预约取消通知** | 取消预约 | Email+站内信 | 取消确认、退款提示 |
| **预约修改通知** | 修改预约 | Email+站内信 | 修改详情、重新确认 |
| **签到提醒** | 预约开始时 | 站内信+大屏 | 签到二维码/位置 |

### 4.5 统计分析体系

| 统计维度 | 数据项 | 展示方式 | 导出格式 |
|---------|--------|---------|---------|
| **空间使用率** | 每个空间的使用时长占比 | 饼图+表格 | CSV/PDF |
| **预约趋势** | 每日/每周/每月预约数量 | 折线图 | CSV |
| **时段分布** | 不同时段的预约热度 | 热力图 | CSV |
| **用户排行** | 用户预约频次排行 | 柱状图 | CSV |
| **收入统计** | 各空间收入汇总 | 表格+总计 | PDF财务报表 |
| **审批效率** | 审批时长、通过率 | 漏斗图 | CSV |
| **设备使用** | 设备借用频次 | 表格 | CSV |

## 五、与现有系统集成

### 5.1 数据迁移策略

```
迁移方案:

Phase 1: 数据模型扩展
├── 保留 meeting_rooms 表
├── 扩展为 space_resources 表（添加类型字段）
├── 保留 meeting_bookings 表
├── 扩展为 space_bookings 表（添加类型相关字段）
└── 新增 pricing_rules、notifications 等表

Phase 2: API兼容过渡
├── 保留 /api/v1/meeting/* 路由（兼容旧前端）
├── 新增 /api/v2/space/* 路由（新版API）
├── v1路由内部转发到v2处理
└── 前端逐步切换到v2

Phase 3: 前端渐进升级
├── 保留原会议室页面（兼容模式）
├── 新增空间服务页面（新版）
├── 通过URL参数或用户选择切换
└── 最终统一到新版页面
```
