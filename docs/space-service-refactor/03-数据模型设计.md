## 一、数据库架构总览

### 1.1 数据库选择

| 维度 | 选择方案 | 说明 |
|------|---------|------|
| **数据库类型** | SQLite（开发）→ PostgreSQL（生产） | 开发环境使用SQLite简化部署，生产环境使用PostgreSQL支持高并发 |
| **ORM框架** | SQLAlchemy 2.0 | 符合FastAPI最佳实践，支持异步操作 |
| **数据迁移** | Alembic | 版本化数据库迁移管理 |
| **连接池** | 生产环境启用连接池 | 最大连接数100，最小连接数10 |

### 2.1 space_types（空间类型定义表）

**用途:** 定义系统支持的所有空间类型及其配置规则

```python
class SpaceType(Base, TimestampMixin):
    """空间类型模型"""
    
    __tablename__ = "space_types"
    
    # 主键
    id = Column(Integer, primary_key=True, index=True)
    
    # 基本信息
    type_code = Column(String(50), unique=True, nullable=False)  # meeting_room, venue, lobby_screen, lobby_booth, vip_dining
    type_name = Column(String(100), nullable=False)  # 会议室、会场、大堂大屏、大堂展位、VIP餐厅
    type_name_en = Column(String(100))  # Meeting Room, Venue, Lobby Screen, Lobby Booth, VIP Dining
    description = Column(Text)  # 类型描述
    
    # 预约规则配置
    min_duration_unit = Column(String(20), nullable=False)  # hour, half_day, full_day, meal_session, day, week
    min_duration_value = Column(Integer, default=1)  # 最小预约时长数值
    max_duration_value = Column(Integer, default=24)  # 最大预约时长数值
    advance_booking_days = Column(Integer, default=0)  # 需提前预约天数
    
    # 容量配置
    min_capacity = Column(Integer, default=1)  # 最小人数
    max_capacity = Column(Integer, default=500)  # 最大人数
    
    # 审批配置
    requires_approval = Column(Boolean, default=False)  # 是否需要审批
    approval_type = Column(String(20))  # none, auto, manual, content_review
    approval_deadline_hours = Column(Integer, default=24)  # 审批截止时间（小时）
    
    # 定金配置
    requires_deposit = Column(Boolean, default=False)  # 是否需要定金
    deposit_percentage = Column(Float, default=0.0)  # 定金比例（0-1）
    deposit_refund_rules = Column(Text)  # JSON格式的定金退还规则
    
    # 设备设施配置
    standard_facilities = Column(Text)  # JSON格式的标准设备列表
    optional_addons = Column(Text)  # JSON格式的可选增值服务
    
    # 状态
    is_active = Column(Boolean, default=True)
    sort_order = Column(Integer, default=0)  # 显示排序
    
    # 图标与颜色
    icon = Column(String(50))  # emoji或图标class
    color_theme = Column(String(20))  # 显示颜色主题
    
    # 关系
    resources = relationship("SpaceResource", back_populates="space_type")
```

### 2.2 space_resources（空间资源表）

**用途:** 管理具体的空间资源实例，替代原有的meeting_rooms表

```python
class SpaceResource(Base, TimestampMixin):
    """空间资源模型"""
    
    __tablename__ = "space_resources"
    
    # 主键
    id = Column(Integer, primary_key=True, index=True)
    
    # 关联空间类型
    type_id = Column(Integer, ForeignKey("space_types.id"), nullable=False, index=True)
    
    # 基本信息
    name = Column(String(100), nullable=False)  # 空间名称
    name_en = Column(String(100))  # 英文名称
    location = Column(String(200))  # 物理位置
    floor = Column(String(20))  # 楼层
    building = Column(String(100))  # 楼栋
    
    # 容量信息
    capacity = Column(Integer, default=10)  # 容纳人数
    capacity_level = Column(String(20))  # small/medium/large
    
    # 设备设施
    facilities = Column(Text)  # JSON格式的设备设施列表
    facilities_status = Column(Text)  # JSON格式的设备状态
    
    # 定价信息
    base_price = Column(Float, default=0.0)  # 基础价格
    price_unit = Column(String(20))  # 计价单位（继承自空间类型）
    member_price = Column(Float, default=0.0)  # 会员价格
    vip_price = Column(Float, default=0.0)  # VIP价格
    
    # 特殊时段定价
    peak_time_price = Column(Float)  # 黄金时段价格（大屏专用）
    off_peak_price = Column(Float)  # 非黄金时段价格（大屏专用）
    
    # 免费额度
    free_hours_per_month = Column(Integer, default=0)  # 每月免费时长（小时）
    
    # VIP餐厅专用字段
    meal_standard_price = Column(Float)  # 标准餐标价格（VIP餐厅）
    meal_vip_price = Column(Float)  # VIP餐标价格（VIP餐厅）
    meal_luxury_price = Column(Float)  # 豪华餐标价格（VIP餐厅）
    
    # 展位专用字段
    booth_size = Column(String(20))  # 展位尺寸：standard(2m×2m), large(4m×4m)
    booth_position = Column(String(20))  # 展位位置：center, edge
    
    # 会场专用字段
    venue_level = Column(String(20))  # 会场等级：small(50-100人), medium(100-200人), large(200-500人)
    setup_time_hours = Column(Integer, default=2)  # 播场时间（小时）
    setup_fee_per_hour = Column(Float)  # 播场费每小时
    
    # 照片与描述
    photos = Column(Text)  # JSON格式的照片URL列表
    description = Column(Text)  # 详细描述
    
    # 状态
    is_active = Column(Boolean, default=True, index=True)
    is_available = Column(Boolean, default=True)  # 是否可预约（维护期间可关闭）
    maintenance_status = Column(String(20), default='normal')  # normal, maintenance, repair
    maintenance_note = Column(Text)  # 维护说明
    
    # 所属办公室（可选）
    office_id = Column(Integer)  # 所属办公室ID（关联water系统）
    
    # 关系
    space_type = relationship("SpaceType", back_populates="resources")
    bookings = relationship("SpaceBooking", back_populates="resource")
```

**数据迁移说明:**
- 原meeting_rooms表数据迁移到space_resources
- 新增type_id字段关联空间类型
- 会议室数据设置type_id=1

```python
class SpaceApproval(Base, TimestampMixin):
    """空间审批模型"""

    # 主键

    booking_no = Column(String(50))  # 预约编号（冗余）
    
    # 审批类型
    approval_type = Column(String(20), nullable=False)  # booking_approval/content_approval/cancel_approval/modify_approval
    approval_stage = Column(String(20))  # first_review/final_review
    
    # 审批内容
    approval_content = Column(Text)  # JSON格式的审批内容摘要
    attachments = Column(Text)  # JSON格式的附件URL列表

    approver_id = Column(Integer)  # 审批人ID
    approver_name = Column(String(100))  # 审批人姓名

    approver_department = Column(String(100))  # 审批人部门

    status = Column(String(20), default='pending', index=True)  # pending/approved/rejected/need_modify
    result = Column(String(20))  # approved/rejected

    submitted_at = Column(DateTime, default=datetime.now)  # 提交时间
    assigned_at = Column(DateTime)  # 分配时间
    reviewed_at = Column(DateTime)  # 审核时间
    approved_at = Column(DateTime)  # 通过时间
    rejected_at = Column(DateTime)  # 拒绝时间
    
    # 审批意见
    approval_notes = Column(Text)  # 审批备注
    rejection_reason = Column(Text)  # 拒绝原因
    modify_suggestions = Column(Text)  # 修改建议（需修改时）
    
    # 审批流转
    prev_approver_id = Column(Integer)  # 上级审批人ID
    next_approver_id = Column(Integer)  # 下级审批人ID
```

## 六、数据迁移方案

### 6.1 迁移策略

```
数据迁移步骤:

Step 1: 创建新表结构
├── 创建space_types表并插入预置数据
├── 创建space_resources表
├── 创建space_bookings表
├── 创建其他辅助表

Step 2: 迁移现有数据
├── meeting_rooms → space_resources
│    ├── 新增type_id=1（会议室类型）
│    ├── 新增容量等级字段
│    └── 其他字段映射
├── meeting_bookings → space_bookings
│    ├── 新增type_code='meeting_room'
│    ├── 状态值映射（pending→pending_approval, confirmed→confirmed）
│    └── 其他字段映射
├── meeting_approvals → space_approvals
├── meeting_payments → space_payments

Step 3: 建立数据关联
├── 更新booking的resource_id关联
├── 更新approval的booking_id关联
├── 更新payment的booking_id关联

Step 4: 数据完整性验证
├── 验证数据数量一致
├── 验证数据关联正确
├── 验证业务流程完整

Step 5: API路由切换
├── 保留旧API路由兼容期（1个月）
├── 新API路由正式启用
└── 旧API路由标记为deprecated
```
