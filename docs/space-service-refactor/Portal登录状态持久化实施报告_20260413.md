# Portal登录状态持久化实施报告

**实施日期:** 2026-04-13  
**实施人员:** 系统架构师  
**项目目标:** 实现Portal登录后访问所有子页面无需重新登录，确保系统安全且便捷好用

---

## 一、问题分析

### 1.1 原问题

用户反馈：在Portal页登录后，进入后续页面需要重新登录。

### 1.2 根本原因

经过深入分析，发现以下关键问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| **Token存储key不一致** | 登录页存储为'token'，但space页面读取为'auth_token' | 认证状态丢失 |
| **缺少Token过期管理** | 没有记录登录时间，无法判断Token是否过期 | 无法自动处理过期 |
| **认证检查不统一** | 各页面认证逻辑各自实现，不一致 | 行为不统一 |
| **缺少Token有效性验证** | 页面加载时不验证Token是否有效 | 可能使用失效Token |

---

## 二、解决方案

### 2.1 创建统一认证管理工具

**文件:** portal/assets/js/auth.js

**核心功能:**

```javascript
const AuthManager = {
    TOKEN_KEY: 'token',              // 统一Token存储key
    TOKEN_EXPIRE_HOURS: 24,          // Token过期时间
    
    getToken()                       // 获取Token
    setToken(token)                  // 设置Token（记录时间戳）
    getUserInfo()                    // 获取用户信息
    isTokenExpired()                 // 检查Token是否过期
    getAuthHeaders()                 // 获取认证请求头
    validateToken()                  // 验证Token有效性（调用API）
    checkAuth(options)               // 完整认证检查流程
    logout()                         // 统一退出登录
}
```

### 2.2 统一认证检查流程

```javascript
async checkAuth(options = {}) {
    const {
        requireAuth = true,      // 是否必须登录
        requireAdmin = false,    // 是否需要管理员权限
        validateToken = true     // 是否验证Token有效性
    } = options;
    
    // 1. 检查Token是否存在
    const token = this.getToken();
    if (!token) {
        if (requireAuth) this.redirectToLogin();
        return null;
    }
    
    // 2. 检查Token是否过期（本地快速检查）
    if (this.isTokenExpired()) {
        this.clearAuth();
        if (requireAuth) this.redirectToLogin();
        return null;
    }
    
    // 3. 验证Token有效性（后端验证）
    if (validateToken) {
        const isValid = await this.validateToken();
        if (!isValid) {
            if (requireAuth) this.redirectToLogin();
            return null;
        }
    }
    
    // 4. 获取用户信息
    const userInfo = this.getUserInfo();
    
    // 5. 检查管理员权限
    if (requireAdmin) {
        const adminRoles = ['super_admin', 'admin', 'office_admin'];
        if (!adminRoles.includes(userInfo.role)) {
            alert('权限不足');
            window.location.href = '/portal/index.html';
            return null;
        }
    }
    
    return userInfo;
}
```

## 三、实施细节

### 3.1 修复Token存储不一致

**修改文件:** portal/admin/space/*.html

```bash
# 批量替换auth_token为token
for file in portal/admin/space/*.html; do
    sed -i '' "s/auth_token/token/g" "$file"
done
```

**效果:** 所有Space管理页面现在使用统一的'token' key。

### 3.2 登录页面记录时间戳

**修改文件:** portal/admin/login.html, space-frontend/login.html

```javascript
// 登录成功后
localStorage.setItem('token', data.access_token);
localStorage.setItem('token_timestamp', new Date().toISOString());  // 新增
localStorage.setItem('userInfo', JSON.stringify(userInfo));
```

**效果:** 可以计算Token是否过期。

### 3.3 Portal首页添加认证检查

**修改文件:** portal/index.html

```javascript
// 添加auth.js引入
<script src="/portal/assets/js/auth.js"></script>

// loadUserData函数开始处
const user = await checkAuth({ 
    requireAuth: false,   // Portal首页允许未登录访问
    validateToken: true   // 验证Token有效性
});

if (user) {
    userInfo.value = { ...user, is_admin: ... };
} else {
    userInfo.value = { id: null, name: '游客', role: 'guest' };
}
```

**效果:** Portal首页自动验证登录状态，无需重新登录。

### 3.4 管理页面添加管理员认证

**修改文件:** portal/admin/index.html, portal/admin/space/*.html

```javascript
// 添加auth.js引入
<script src="/portal/assets/js/auth.js"></script>

// onMounted函数
onMounted(async () => {
    const user = await checkAuth({ 
        requireAuth: true,      // 必须登录
        requireAdmin: true,     // 需要管理员权限
        validateToken: true     // 验证Token有效性
    });
    
    if (user) {
        currentUser.value = user;
        await loadDashboard();
    }
});
```

**效果:** 管理页面自动检查登录和管理员权限。

### 5.1 单点登录（SSO）

```
用户登录
    ↓
localStorage.setItem('token', token)
localStorage.setItem('token_timestamp', timestamp)
localStorage.setItem('userInfo', userInfo)
    ↓
访问任意页面
    ↓
checkAuth()自动验证
    ↓
无需重新登录 ✅
```

### 5.2 自动Token验证

| 验证层级 | 验证方式 | 执行时机 | 耗时 |
|----------|---------|---------|------|
| **本地检查** | 检查Token是否存在 | 页面加载 | 1ms |
| **过期检查** | 计算时间戳是否超过24小时 | 页面加载 | 1ms |
| **后端验证** | 调用/auth/profile API | 页面加载 | 50-200ms |

**效果:** 三层验证确保安全性，本地检查避免频繁请求。

### 5.3 Token过期处理

```javascript
// 本地快速检查（不请求后端）
const hoursPassed = (now - loginTime) / (1000 * 60 * 60);
if (hoursPassed >= 24) {
    clearAuth();
    redirectToLogin();
}
```

**效果:** Token过期自动跳转登录页，无需等待后端响应。

| 安全措施 | 实现方式 | 效果 |

## 七、使用说明

### 7.1 页面引入auth.js

```html
<!DOCTYPE html>
<html>
<head>
    <!-- 在<head>部分引入 -->
    <script src="/portal/assets/js/auth.js"></script>
</head>
<body>
    ...
</body>
</html>
```

### 7.2 页面初始化调用checkAuth

**普通页面（允许未登录访问）:**

```javascript
onMounted(async () => {
    const user = await checkAuth({ 
        requireAuth: false,   // 不强制登录
        validateToken: true   // 验证Token有效性
    });
    
    if (user) {
        // 已登录用户
        console.log('用户已登录:', user.name);
    } else {
        // 未登录或Token无效
        console.log('用户未登录');
    }
});
```

**管理页面（必须管理员登录）:**

```javascript
onMounted(async () => {
    const user = await checkAuth({ 
        requireAuth: true,      // 必须登录
        requireAdmin: true,     // 必须管理员
        validateToken: true     // 验证Token有效性
    });
    
    if (user) {
        // 管理员用户，加载管理数据
        await loadDashboard();
    }
});
```

### 7.3 API请求使用认证头

```javascript
const response = await fetch('/api/v1/some-endpoint', {
    method: 'GET',
    headers: getAuthHeaders()  // 自动添加Authorization头
});
```

### 7.4 退出登录

```javascript
// 退出按钮点击事件
const handleLogout = async () => {
    await logout();  // 自动清除Token并跳转登录页
};
```
