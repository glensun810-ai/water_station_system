## 二、统一登录架构设计

### 2.1 设计原则

1. **单一入口原则** - 所有登录使用统一的API端点和页面
2. **角色自动识别** - 登录后根据用户role自动跳转到对应页面
3. **统一认证机制** - 全系统使用JWT认证
4. **统一会话管理** - 统一的Token管理和黑名单机制
5. **安全标准统一** - 符合OAuth 2.0、OWASP、NIST安全规范

### 2.2 统一登录流程

```
用户访问登录页面
    ↓
输入用户名/密码
    ↓
调用统一认证API (/api/v1/system/auth/login)
    ↓
API验证用户信息
    ├── 检查登录尝试限制
    ├── 验证密码
    ├── 检查用户状态
    ├── 创建JWT Token
    └── 返回Token + 用户信息
    ↓
前端存储Token（localStorage）
    ↓
根据用户role跳转:
    ├── super_admin → /portal/admin/index.html（系统管理后台）
    ├── admin → /portal/admin/index.html（管理后台）
    ├── office_admin → /portal/admin/index.html（管理后台）
    └── user/staff → /portal/index.html（用户端首页）
    └── guest → /portal/index.html（用户端首页）
```

### 2.3 统一登录页面设计

**推荐方案：使用 portal/admin/login.html 作为统一登录入口**

**理由:**
1. 已完整实现JWT认证
2. 已实现安全措施（登录尝试限制、密码强度验证）
3. 已实现角色自动识别跳转
4. 已实现记住用户名功能
5. 已实现登录日志记录

**整合方案:**

| 功能 | 当前状态 | 整合方案 |
|------|---------|---------|
| 用户端登录入口 | space-frontend/login.html | 重定向到统一登录页面 |
| 管理端登录入口 | portal/admin/login.html | 作为统一登录入口 |
| 登录API | /api/v1/system/auth/login | 统一使用此API |

async def get_admin_user(
    current_user: User = Depends(get_current_user_required)
) -> User:
    """获取管理员用户"""
    if current_user.role not in ['admin', 'super_admin', 'office_admin']:
        raise HTTPException(403, "需要管理员权限")
    return current_user

async def get_super_admin_user(
    current_user: User = Depends(get_current_user_required)
) -> User:
    """获取超级管理员"""

        raise HTTPException(403, "需要超级管理员权限")

## 四、统一前端登录页面

### 4.1 登录页面统一方案

**方案：portal/admin/login.html 作为统一入口**

**跳转逻辑:**

```javascript
// 登录成功后根据角色跳转
const handleLoginSuccess = (userData) => {
    localStorage.setItem('token', userData.access_token);
    localStorage.setItem('userInfo', JSON.stringify(userData.user));
    
    // 根据角色跳转
    const role = userData.user.role;
    
    if (role === 'super_admin' || role === 'admin') {
        // 系统管理员 → 系统管理后台
        window.location.href = '/portal/admin/index.html';
    } else if (role === 'office_admin') {
        // 办公室管理员 → 管理后台
        window.location.href = '/portal/admin/index.html';
    } else if (role === 'user' || role === 'staff') {
        // 内部员工 → 用户端首页
        window.location.href = '/portal/index.html';
    } else if (role === 'guest') {
        // 外部访客 → 用户端首页
        window.location.href = '/portal/index.html';
    }
};
```

## 五、统一会话管理

### 5.1 Token存储统一

**全系统统一使用localStorage:**

| Key | Value | 说明 |
|-----|-------|------|
| token | JWT access_token | 认证Token |
| userInfo | JSON.stringify(user) | 用户信息 |
| rememberedUsername | username | 记住用户名 |

### 5.2 Token验证统一

**所有页面统一验证逻辑:**

```javascript
// 检查登录状态
const checkLoginStatus = () => {
    const token = localStorage.getItem('token');
    if (!token) {
        // 未登录，跳转登录页
        window.location.href = '/portal/admin/login.html';
        return false;
    }
    
    // 可选：验证Token有效性
    verifyToken(token).then(valid => {
        if (!valid) {
            localStorage.clear();
            window.location.href = '/portal/admin/login.html';
        }
    });
    
    return true;
};
```

### 5.3 退出登录统一

**全系统统一退出逻辑:**

```javascript
const handleLogout = async () => {
    const token = localStorage.getItem('token');
    
    try {
        // 调用退出API（将Token加入黑名单）
        await fetch('/api/v1/system/auth/logout', {
            method: 'POST',
            headers: {
                'Authorization': `Bearer ${token}`
            }
        });
    } catch (e) {
        console.error('退出API调用失败:', e);
    }
    
    // 清除本地存储
    localStorage.removeItem('token');
    localStorage.removeItem('userInfo');
    
    // 跳转登录页
    window.location.href = '/portal/admin/login.html';
};
```
