## 问题诊断

### 问题表现
用户访问 `http://127.0.0.1:8000/portal/admin/meeting/rooms.html` 时，会议室管理页面为空。

### 问题根因

1. **端口配置错误**
   - 用户访问8000端口，但API服务运行在8008端口
   - 管理页面硬编码使用旧架构端口（8001）

2. **API路径错误**
   - 使用旧架构路径：`/api/meeting/rooms`
   - 新架构路径：`/api/v1/meeting/rooms`

3. **缺少认证Header**
   - 部分API请求未添加Authorization header
   - 导致API返回"未登录或登录已过期"

---

## 修复内容

### 修复的页面（3个）

1. **portal/admin/meeting/rooms.html** - 会议室管理页面
2. **portal/admin/meeting/bookings.html** - 预约管理页面
3. **portal/admin/index.html** - 管理后台首页

### 修复的具体内容

#### 1. API端口统一
```javascript
// 旧配置（错误）
const MEETING_PORT = '8001';
const API_BASE = `${protocol}//${hostname}:${MEETING_PORT}/api/meeting`;

// 新配置（正确）
const API_BASE = `${window.location.protocol}//${window.location.hostname}:${window.location.port || '8008'}/api/v1`;
```

#### 2. API路径标准化
```javascript
// 旧路径（错误）
fetch(`${API_BASE}/rooms`)
fetch(`${API_BASE}/bookings`)

// 新路径（正确）
fetch(`${API_BASE}/meeting/rooms`)
fetch(`${API_BASE}/meeting/bookings`)
```

#### 3. 添加认证Header
```javascript
// 旧请求（错误）
fetch(`${API_BASE}/meeting/rooms`)

// 新请求（正确）
const token = localStorage.getItem('token');
fetch(`${API_BASE}/meeting/rooms`, {
    headers: { 'Authorization': `Bearer ${token}` }
})
```
