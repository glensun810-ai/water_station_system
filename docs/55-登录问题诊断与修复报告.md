## 一、问题诊断

### 1.1 用户报告的问题

```
登录页提示报错：
- Access to fetch at 'http://localhost:8000/api/auth/login' has been blocked by CORS policy
- Response to preflight request doesn't pass access control check
- Failed to fetch
```

### 1.2 根本原因分析

#### 问题1: API端点错误
- ❌ **原请求**: `/api/auth/login`
- ✅ **实际端点**: `/api/admin/login`

#### 问题2: 请求字段错误
- ❌ **原请求字段**: `name: 'admin'`
- ✅ **实际字段**: `username: 'admin'`

#### 问题3: 硬编码端口
- ❌ **原配置**: `http://localhost:8000/api`
- ✅ **修复后**: 动态获取当前端口

## 三、修复方案

### 3.1 修复1: API端点
**文件**: `Service_WaterManage/frontend/login.html`

```javascript
// ❌ 修复前
const response = await fetch(`${API_BASE}/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        name: this.username,
        password: this.password
    })
});

// ✅ 修复后
const response = await fetch(`${API_BASE}/admin/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        username: this.username,
        password: this.password
    })
});
```

### 3.3 修复3: 动态端口配置
**文件**: `Service_WaterManage/frontend/login.html`

```javascript
// ❌ 修复前
const API_BASE = localStorage.getItem('API_BASE') || (isLocalDev ? 'http://localhost:8000/api' : 'https://jhw-ai.com/api');

// ✅ 修复后
const protocol = window.location.protocol;
const hostname = window.location.hostname;
const port = window.location.port || (protocol === 'https:' ? '443' : '80');
const API_BASE = `${protocol}//${hostname}:${port}/api`;
```
