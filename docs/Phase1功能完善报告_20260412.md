### 3. ✅ 完善统计API（6个新端点）

**新增端点**：`apps/api/v1/meeting.py`

#### 3.1 使用率统计API
```
GET /api/v1/meeting/stats/usage-rate
```
**功能**：
- 计算公式：实际使用小时/可用总小时 * 100
- 支持按会议室筛选
- 支持自定义日期范围
- 默认统计最近7天

**返回数据**：
```json
{
  "start_date": "2026-04-05",
  "end_date": "2026-04-12",
  "room_id": null,
  "total_used_hours": 45.5,
  "total_available_hours": 1050,
  "usage_rate": 4.33,
  "booking_count": 10
}
```

#### 3.2 周统计API
```
GET /api/v1/meeting/stats/weekly
```
**功能**：
- 本周一至周日统计
- 预约数、确认数、总收入
- 日均预约数计算

**返回数据**：
```json
{
  "week_start": "2026-04-07",
  "week_end": "2026-04-13",
  "booking_count": 25,
  "confirmed_count": 20,
  "total_revenue": 2500.00,
  "avg_daily_bookings": 3.6
}
```

#### 3.3 月统计API
```
GET /api/v1/meeting/stats/monthly?month=4&year=2026
```
**功能**：
- 预约数、确认数、完成数、取消数
- 总收入、待收入
- 日均预约数、完成率计算

**返回数据**：
```json
{
  "month": 4,
  "year": 2026,
  "days_in_month": 30,
  "booking_count": 100,
  "confirmed_count": 80,
  "completed_count": 70,
  "cancelled_count": 10,
  "total_revenue": 8500.00,
  "pending_revenue": 1500.00,
  "avg_daily_bookings": 3.3,
  "completion_rate": 80.0
}
```

#### 3.4 收入统计API
```
GET /api/v1/meeting/stats/revenue?start_date=2026-04-01&end_date=2026-04-30
```
**功能**：
- 总收入、待收入、取消收入
- 已支付预约数
- 平均每单收入

**返回数据**：
```json
{
  "start_date": "2026-04-01",
  "end_date": "2026-04-30",
  "total_revenue": 8500.00,
  "pending_revenue": 1500.00,
  "cancelled_revenue": 500.00,
  "paid_booking_count": 70,
  "avg_revenue_per_booking": 120.71
}
```

#### 3.5 趋势数据API
```
GET /api/v1/meeting/stats/trend?days=7
```
**功能**：
- 每日预约数、收入数据
- 用于ECharts图表展示
- 默认7天，可自定义

**返回数据**：
```json
{
  "start_date": "2026-04-06",
  "end_date": "2026-04-12",
  "days": 7,
  "trend_data": [
    {
      "date": "2026-04-06",
      "booking_count": 3,
      "revenue": 450.00
    },
    // ...更多日期
  ]
}
```

#### 3.6 今日统计API（已存在，保留）
```
GET /api/v1/meeting/stats/today
```

### 4. ✅ 修改预约功能PUT接口

**新增端点**：
```
PUT /api/v1/meeting/bookings/{booking_id}
```

**功能实现**：
1. **支持修改项**
   - 预约日期
   - 开始时间、结束时间
   - 会议主题
   - 参会人数
   - 预约人电话、部门

2. **智能验证**
   - 时间格式验证（HH:MM）
   - 结束时间必须晚于开始时间
   - 时长范围验证（30分钟-8小时）
   - 实时冲突检测（排除当前预约）
   - 容量验证（参会人数≤会议室容量）

3. **自动更新**
   - 重新计算时长和费用
   - 更新total_fee、actual_fee

4. **权限控制**
   - 普通用户只能修改自己的预约
   - 只能修改待审批或已确认状态的预约
   - 管理员可修改任意预约

**Pydantic Schema**：
```python
class MeetingBookingUpdate(BaseModel):
    booking_date: Optional[date] = None
    start_time: Optional[str] = None
    end_time: Optional[str] = None
    meeting_title: Optional[str] = None
    attendees_count: Optional[int] = None
    user_phone: Optional[str] = None
    department: Optional[str] = None
```
