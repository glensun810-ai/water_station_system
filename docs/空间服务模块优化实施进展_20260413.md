## 三、待完成工作

### 3.1 API端点优化

**需新增的API端点**（按统计维度）：

1. **按办公室统计API**
```
GET /api/v2/space/stats/office/{office_id}
- 参数：start_date, end_date
- 返回：办公室整体使用情况、成员排行、费用归属、免费额度使用情况
- 权限：管理员 + 该办公室成员
```

2. **按个人统计API**
```
GET /api/v2/space/stats/user/{user_id}
- 参数：start_date, end_date
- 返回：个人预约统计、常用空间、费用汇总、支付记录
- 权限：用户本人 + 管理员
```

3. **全局统计API（管理员）**
```
GET /api/v2/space/stats/global
- 参数：start_date, end_date, dimension (office/person)
- 返回：所有办公室汇总或所有个人汇总
- 权限：仅管理员
```

**需修改的现有API**：

- `POST /api/v1/meeting/bookings` 
  - 移除参数：user_type, office_id, user_name（手动传入）
  - 改为：从Authorization token自动获取user对象，自动填充
  
- `GET /api/v1/meeting/bookings`
  - 添加统计维度参数：`dimension=office|person`
  - 按dimension返回不同的数据结构

### 3.2 数据模型扩展

**User模型需扩展字段**：

```python
class User(Base):
    # 新增字段
    office_id = Column(Integer, ForeignKey("offices.id"))  # 所属办公室
    computed_user_type = property(lambda self: 'internal' if self.office_id else 'external')
    free_hours_monthly = Column(Integer, default=2)  # 每月免费小时
```

## 四、关键决策记录

### 4.1 设计决策

| 决策项 | 决策内容 | 决策依据 |
|-------|---------|---------|
| **用户类型识别方式** | 从登录账户自动识别 | Microsoft Outlook标准实践 |
| **统计维度选择** | 内部按办公室，外部按个人 | 企业管理需求 + WeWork标准 |
| **免费额度管理** | 每月分配，按办公室汇总 | 企业会员制度需求 |
| **预约流程简化** | 从7步降至4步 | 用户体验优化43%目标 |

### 4.2 技术决策

| 决策项 | 决策内容 | 决策依据 |
|-------|---------|---------|
| **API版本** | 保持v1兼容，新增v2 | 系统平滑过渡 |
| **统计维度实现** | API参数dimension控制 | 单一API多维度返回 |
| **前端状态管理** | localStorage + computed property | 无需引入复杂状态库 |
| **权限控制** | API层统一拦截 | 安全性优先 |
