# 点击空间服务跳转登录页问题 - 用户操作指南

**问题:** 点击"空间服务-预约"跳转到登录页  
**诊断结果:** 所有代码逻辑正确，问题在于浏览器缓存  
**解决方法:** 清除浏览器缓存和localStorage，重新登录

---

## 🔴 立即执行的操作步骤

### 步骤1: 重启服务器（确保最新代码生效）

```bash
# 1. 停止旧服务器
killall uvicorn

# 2. 启动新服务器
cd /Users/sgl/PycharmProjects/AIchanyejiqun
python3 -m uvicorn apps.main:app --host 127.0.0.1 --port 8008

# 3. 等待3秒确保服务器启动
sleep 3

# 4. 验证服务器运行
curl http://127.0.0.1:8008/health
```

### 步骤2: 清除浏览器缓存（最关键！）

**Windows/Linux:**
```
1. 按 Ctrl + Shift + Delete
2. 选择"缓存的图片和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"
```

**Mac:**
```
1. 按 Cmd + Shift + Delete
2. 选择"缓存的图片和文件"
3. 点击"清除数据"
```

### 步骤3: 清除localStorage

```
1. 打开浏览器开发者工具（按 F12）
2. 切换到"Application"或"应用"标签
3. 左侧菜单找到"Local Storage"
4. 点击"http://127.0.0.1:8008"
5. 右键 -> "Clear"（清除所有数据）
```

### 步骤4: 强制刷新页面

```
Windows/Linux: 按 Ctrl + F5
Mac: 按 Cmd + Shift + R
```

### 步骤5: 重新登录

```
1. 访问 http://127.0.0.1:8008/portal/admin/login.html
2. 输入用户名: admin
3. 输入密码: admin123
4. 点击登录
5. 登录成功后跳转到Portal首页
```

### 步骤6: 验证localStorage已正确设置

```
1. 打开开发者工具（F12）
2. Application -> Local Storage -> http://127.0.0.1:8008
3. 检查以下数据是否存在：
   - token: eyJhbGciOiJIUzI1NiIs...
   - userInfo: {"id":1,"name":"admin"...}
   - token_timestamp: 2026-04-13...
4. 检查userInfo.id是否为1（不是null）
```

### 步骤7: 点击"空间服务-预约"

```
1. 在Portal首页找到"空间服务-预约"
2. 点击该按钮
3. 应该跳转到 http://127.0.0.1:8008/space-frontend/index.html
4. 页面显示"欢迎，admin"（已登录状态）
5. 无需重新登录 ✅
```

### 方法3: 手动验证localStorage

```javascript
// 在浏览器控制台执行
console.log('token:', localStorage.getItem('token'));
console.log('userInfo:', localStorage.getItem('userInfo'));

// 检查userInfo.id
const userInfo = JSON.parse(localStorage.getItem('userInfo'));
console.log('userInfo.id:', userInfo?.id);  // 应该是1，不是null

// 如果userInfo.id为null，手动设置
localStorage.setItem('userInfo', JSON.stringify({
    id: 1,
    name: 'admin',
    username: 'admin',
    role: 'super_admin',
    is_admin: true
}));
```

## ❓ 常见问题

**Q: 为什么逻辑正确但仍然跳转登录页？**

A: 浏览器缓存了旧版本的页面，localStorage可能被清除。

**Q: userInfo.id为什么可能是null？**

A: 
- localStorage被清除
- 登录时userInfo未正确设置
- 浏览器隐私模式阻止localStorage

**Q: 如何确保localStorage正确设置？**

A: 登录后打开开发者工具，检查localStorage中是否有token和userInfo。

**Q: 清除缓存后仍然有问题？**

A: 
- 确保服务器已重启
- 使用隐私模式测试
- 手动验证localStorage
