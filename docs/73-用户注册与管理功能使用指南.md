## 一、用户注册流程

### 内部用户注册

1. **访问注册页面**
   - 打开: http://localhost:8000/portal/register.html

2. **选择用户类型**
   - 点击"内部用户"卡片

3. **填写注册信息**
   - 用户名：2-50字符，支持中文、字母、数字、下划线
   - 密码：至少6位
   - 所属办公室：可选

4. **提交注册**
   - 点击"注册"按钮
   - 提示："注册成功，请等待管理员审核激活"

5. **等待激活**
   - 管理员在用户管理页面激活账号
   - 激活后可正常登录使用

### 外部用户注册

1. **访问注册页面**
   - 打开: http://localhost:8000/portal/register.html

2. **选择用户类型**
   - 点击"外部用户"卡片

3. **填写注册信息**
   - 用户名：2-50字符
   - 密码：至少6位
   - 手机号：必填，11位手机号
   - 邮箱：可选
   - 公司名称：可选

4. **提交注册**
   - 点击"注册"按钮
   - 提示："注册成功"
   - 账号自动激活，可立即登录

5. **使用服务**
   - 登录后可购买会员
   - 可使用会议室服务

## 二、管理员激活用户流程

### 使用管理页面

1. **登录管理后台**
   - 访问: http://localhost:8000/portal/admin/login.html
   - 使用管理员账号登录

2. **进入用户管理**
   - 点击左侧菜单"用户管理"

3. **查找待激活用户**
   - 查看用户列表
   - 状态为"待激活"的用户

4. **激活用户**
   - 点击用户操作菜单
   - 选择"激活"选项
   - 确认激活

5. **验证激活**
   - 用户状态变为"已激活"
   - 用户可正常登录

### 使用测试页面（推荐）

1. **访问测试页面**
   - 打开: http://localhost:8000/portal/test-user-management.html

2. **查看待激活用户**
   - 点击"待激活"筛选按钮
   - 查看所有待激活用户

3. **一键激活**
   - 找到目标用户
   - 点击"激活"按钮
   - 确认激活

4. **查看结果**
   - 用户状态更新为"已激活"
   - 统计数据自动更新

## 三、API使用示例

### 1. 用户注册

```bash
# 内部用户注册
curl -X POST http://localhost:8000/api/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "内部用户001",
    "password": "password123",
    "user_type": "internal"
  }'

# 外部用户注册
curl -X POST http://localhost:8000/api/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "外部用户001",
    "password": "password123",
    "user_type": "external",
    "phone": "13800138000",
    "email": "user@example.com",
    "company": "测试公司"
  }'
```

### 2. 用户激活

```bash
# 单用户激活
curl -X POST http://localhost:8000/api/users/16/activate

# 批量激活
curl -X POST http://localhost:8000/api/users/batch/activate \
  -H "Content-Type: application/json" \
  -d '[15, 16, 17]'
```

### 3. 查询用户列表

```bash
# 查询所有用户
curl http://localhost:8000/api/users

# 按用户类型筛选
curl "http://localhost:8000/api/users?user_type=external"

# 按激活状态筛选
curl "http://localhost:8000/api/users?is_active=0"

# 搜索用户
curl "http://localhost:8000/api/users?search=张三"
```

### 4. 查询统计信息

```bash
curl http://localhost:8000/api/users/stats/overview
```

返回示例：
```json
{
  "total": 10,
  "active": 8,
  "inactive": 2,
  "internal_users": 10,
  "external_users": 0,
  "super_admins": 1,
  "admins": 3,
  "office_admins": 1,
  "users": 5
}
```

## 四、常见问题

### Q1: 内部用户注册后无法登录？
**A**: 内部用户需要管理员激活后才能登录。请管理员在用户管理页面激活账号。

### Q2: 外部用户注册后是否需要激活？
**A**: 不需要。外部用户注册后自动激活，可立即登录使用。

### Q3: 如何区分内部用户和外部用户？
**A**: 
- 内部用户：企业员工，需要管理员审核激活
- 外部用户：外部访客，注册即可使用，可购买会员

### Q4: 用户激活后可以取消吗？
**A**: 可以。管理员可以在用户管理页面停用用户（设置is_active=0）。

### Q5: 忘记密码怎么办？
**A**: 目前需要联系管理员重置密码。后续会实现找回密码功能。
