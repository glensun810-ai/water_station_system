## 一、功能审核发现问题

### 1.1 核心问题诊断

**问题现象**：
- 用户访问领水页面，看不到"选择办公室"功能
- 点击"确认领水"无响应或报错
- "我的记录"页面加载失败

**根因分析**：

#### 问题1：办公室选择功能失效

**原因**：
```javascript
// 旧的 displayOffices 计算属性（有bug）
displayOffices() { 
    if (this.userOffices.length > 0) {
        return this.offices.filter(o => 
            userOfficeIds.includes(o.id) && 
            (this.officeTab === 'common' ? o.is_common === 1 : o.is_common !== 1)
        );
    }
    return this.offices.filter(o => 
        this.officeTab === 'common' ? o.is_common === 1 : o.is_common !== 1
    ); 
}
```

**问题点**：
1. API返回的办公室数据**没有 `is_common` 字段**
2. 所有办公室的 `is_common` 都是 `undefined`
3. 过滤条件 `o.is_common === 1` 永远为 `false`
4. 导致 `displayOffices` 返回空数组
5. 用户看不到任何办公室选项

#### 问题2：管理员首次访问逻辑错误

**原因**：
```javascript
// 旧逻辑（错误）
if (userInfo.role === 'admin' || userInfo.role === 'super_admin') {
    this.showFirstTimeGuide = true;  // ❌ 显示引导，但无法设置办公室
}
```

**影响**：
- 管理员首次访问被强制显示引导
- 但管理员不需要引导，应该直接看到所有办公室
- 用户被困在引导流程中无法继续

#### 问题3：前后端文件不一致

**现状**：
- 已更新：`/portal/water/index.html`（新版）
- 未更新：`/Service_WaterManage/frontend/index.html`（旧版）
- 用户访问：`http://127.0.0.1:8008/water/index.html` → 旧版页面 ❌

**导致**：
- 用户看到的是未修复的旧页面
- 新功能无法生效

```

└─ 无效/过期：返回 None
↓
业务逻辑处理
├─ current_user 存在：记录创建者信息
└─ current_user 为 None：可能允许访客模式或要求登录
```

#### 优化3：优惠计算实时显示

**实现**：
```javascript
// 计算"买N赠M"优惠
totalGiftQty() {
    return this.products.reduce((sum, p) => {
        if (!this.cart[p.id]) return sum;
        const qty = this.cart[p.id];
        if (p.promo_threshold && p.promo_gift && qty >= p.promo_threshold) {
            const cycle = p.promo_threshold + p.promo_gift;
            const cycles = Math.floor(qty / cycle);
            const remainder = qty % cycle;
            let gift = cycles * p.promo_gift;
            if (remainder > p.promo_threshold) {
                gift += Math.min(remainder - p.promo_threshold, p.promo_gift);
            }
            sum += gift;
        }
        return sum;
    }, 0);
}
```

**效果**：
- ✅ 实时计算赠送数量
- ✅ 显示优惠金额
- ✅ 用户清楚知道实际应付金额
