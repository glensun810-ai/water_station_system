## 二、用户生命周期设计

```
注册 → 审核（内部用户）→ 激活 → 使用 → 管理 → 停用/删除
  ↓
自动激活（外部用户）
```

### 用户状态流转

#### 内部用户
1. 用户注册 → 状态：待激活（is_active=0）
2. 管理员审核 → 状态：已激活（is_active=1）
3. 管理员可停用 → 状态：已停用（is_active=0）

#### 外部用户
1. 用户注册 → 状态：已激活（is_active=1）
2. 可购买会员
3. 管理员可停用

```python
@router.post("/{user_id}/activate")
def activate_user(user_id: int, db: Session = Depends(get_db)):
    """激活用户"""
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(status_code=404, detail="用户不存在")
    
    user.is_active = 1
    user.activated_at = datetime.now()
    user.activated_by = get_current_user_id()  # 记录激活人
    db.commit()
    
    # 发送激活通知（可选）
    send_activation_notification(user)
    
    return {"success": True, "message": "用户已激活"}
```

#### 2. 批量激活API
```python
@router.post("/batch/activate")
def batch_activate_users(user_ids: List[int], db: Session = Depends(get_db)):
    """批量激活用户"""
    activated_count = 0
    for user_id in user_ids:
        user = db.query(User).filter(User.id == user_id).first()
        if user and user.is_active == 0:
            user.is_active = 1
            user.activated_at = datetime.now()
            activated_count += 1
    
    db.commit()
    return {
        "success": True,
        "activated_count": activated_count,
        "message": f"成功激活 {activated_count} 个用户"
    }
```
